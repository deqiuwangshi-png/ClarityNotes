  └────────────────────┬─────────────────────────────────────────┘

  ┌────────────────────▼─────────────────────────────────────────┐
  │  数 据 层  (Supabase PostgreSQL) — 四表分离                    │

### 核心原则

| 原则 | 说明 |
|------|------|
| **UI 零副作用** | 禁止 `window.prompt`/`confirm`，统一 `uiStore` + `ConfirmDialog` |
| **用户来源统一** | `UserSection` 内部 `useAuth().user`，不再通过 props 传递 |
| **组件拆分** | `SidebarLayout` → `SidebarLayout` + `SidebarContent` + `CollapsedTopBar` |
| **三表分离** | `file_tree` + `documents` + `trash`，FK CASCADE 保证一致性 |
| **RLS 安全** | 所有表 ENABLE ROW LEVEL SECURITY，按 `user_id` 隔离 |

## 二、数据库设计

### 2.1 建表脚本唯一来源：`docs/1.sql`

### 2.2 表关系图

```
auth.users
    │ 1:1
    ▼
public.profiles (id = auth.users.id)
    │ 1:N
    ▼
public.file_tree (user_id FK → profiles.id)
    │ 1:1 (file_id UNIQUE)
    ▼
public.documents (file_id FK → file_tree.id, ON DELETE CASCADE)

public.trash (user_id FK → profiles.id, original_id = 原 file_tree.id)
```

### 2.3 核心表结构

```sql
-- file_tree: 邻接表树结构
CREATE TABLE public.file_tree (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id       UUID NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
    parent_id     UUID REFERENCES public.file_tree(id) ON DELETE CASCADE,
    name          TEXT NOT NULL DEFAULT '未命名',
    type          node_type NOT NULL,
    level         SMALLINT NOT NULL DEFAULT 0,
    sort_order    INTEGER NOT NULL DEFAULT 0,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- documents: 文档内容（file_id UNIQUE）
CREATE TABLE public.documents (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    file_id       UUID NOT NULL UNIQUE REFERENCES public.file_tree(id) ON DELETE CASCADE,
    content       JSONB NOT NULL DEFAULT '{"type":"doc","content":[]}'::jsonb,
    word_count    INTEGER NOT NULL DEFAULT 0,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- trash: 回收站（30 天自动过期）
CREATE TABLE public.trash (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
    original_id         UUID NOT NULL,
    name                TEXT NOT NULL,
    type                node_type NOT NULL,
    parent_id           UUID,
    original_level      SMALLINT NOT NULL DEFAULT 0,
    content             JSONB,
    word_count          INTEGER DEFAULT 0,
    original_created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    original_updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    expires_at          TIMESTAMPTZ NOT NULL DEFAULT (now() + INTERVAL '30 days')
);
```

## 四、状态层 — Zustand Store

### 4.1 fileTreeStore 数据流

```
loadTree()
  → fileTreeRepo.getTree()
    → supabase.from('file_tree').select('*, documents(content, word_count)')
      → buildTree(flatNodes) → TreeNode[]
        → getTree() 公开 selector 供外部读取

createFile(parentId)
  → supabaseAuthRepo.getSession() → userId
  → fileTreeRepo.insertNode() → UUID
  → addChildToTree(localTree, parentId, newNode)  // 本地更新

commitRename(nodeId, newName)
  → renameNodeService(tree, nodeId, newName)  // 乐观更新
  → fileTreeRepo.updateNode(nodeId, { name })  // Supabase
    → 失败 → 回滚

deleteNode(nodeId)
  → collectDescendantIds(tree, nodeId)  // 收集子孙 ID
  → moveToTrash(tree, nodeId)           // 乐观移除
  → fileTreeRepo.moveToTrash(ids)       // Supabase
    → 失败 → loadTree() 回滚

getTree() / getSelectedNode()          // ★ 公开 selector
```

### 4.2 editorStore 自动保存

```
setContent(doc) / setTitle(title)
  → set({ isDirty: true, isSaved: false })
  → scheduleSave() → 2s 防抖

performSave()
  → editorService.saveDocument({ nodeId, title, content, wordCount })
    → fileTreeRepo.updateNode(nodeId, { name: title })
    → fileTreeRepo.upsertDocument(nodeId, content, wordCount)
```

### 4.3 trashStore 操作

```
loadTrash()
  → trashRepo.getTrash()

restoreItem(originalId)
  → trashRepo.restoreItem(originalId)
  → ★ 内部自动 fileTreeStore.loadTree()  # 不再需要 TrashPage 手动调用

batchRestore()
  → Promise.all(trashRepo.restoreItem)
  → ★ 内部自动 fileTreeStore.loadTree()
```

### 4.4 uiStore — 确认对话框

```
openConfirm({ title, message, confirmLabel, variant })
  → 返回 Promise<boolean>
  → <ConfirmDialog /> 渲染 → 用户选择 → resolve(true/false)
```

## 七、数据库修正脚本（需在 Supabase SQL Editor 执行）

如果用户已执行过旧版 `1.sql`（不含根文件夹触发器），需执行以下补充脚本：

```sql
-- 1. 更新触发器函数（新增根文件夹创建）
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, uid, email, full_name, avatar)
  VALUES (
    NEW.id,
    substring(md5(random()::text), 1, 8),
    NEW.email,
    COALESCE(NEW.raw_user_meta_data->>'full_name', ''),
    COALESCE(NULLIF(left(COALESCE(NEW.raw_user_meta_data->>'full_name', ''), 1), ''), '用')
  );
  INSERT INTO public.file_tree (user_id, parent_id, name, type, level, sort_order)
  VALUES (NEW.id, NULL, '我的文档', 'folder', 1, 0);
  RETURN NEW;
END;
$$ language 'plpgsql' SECURITY DEFINER;

-- 2. 为已有用户补充根文件夹
INSERT INTO public.file_tree (user_id, parent_id, name, type, level, sort_order)
SELECT id, NULL, '我的文档', 'folder', 1, 0
FROM public.profiles p
WHERE NOT EXISTS (
  SELECT 1 FROM public.file_tree ft WHERE ft.user_id = p.id AND ft.parent_id IS NULL
);
```
