## 一、七层架构总览

```
  ═══════════════════════════════════════════════════════════════════
  服务端入口 — proxy.ts（每次请求先经过，完成 Session 刷新 + 路由守卫）

  ┌──────────────────────────────────────────────────────────────┐
  │  ＵＩ 层  (components/ + app/)                               │
  │  只负责渲染和事件派发，不包含业务逻辑、数据存取                 │
  │  禁止 window.prompt/confirm，统一使用 uiStore + ConfirmDialog │
  │  示例：FileTree, TreeItem, Editor, FolderView, UserSection   │
  └────────────────────┬─────────────────────────────────────────┘
                       │ 调用 Zustand stores / useAuth()
  ┌────────────────────▼─────────────────────────────────────────┐
  │  状 态 层  (store/ + lib/auth-context.tsx + hooks/)          │
  │  Zustand stores 管理全局状态，auth-context 管理认证状态       │
  │  调用 services（业务逻辑）和 repositories（数据存取）           │
  │  示例：fileTreeStore, editorStore, trashStore, searchStore   │
  │        uiStore, useAuth()                                    │
  └────────────────────┬─────────────────────────────────────────┘
                       │ 调用
  ┌────────────────────▼─────────────────────────────────────────┐
  │  业 务 逻 辑 层  (lib/services/)                             │
  │  纯函数，实现所有业务规则：校验、树操作、搜索匹配、保存编排     │
  │  不依赖 React、DOM、状态管理，给定输入必定返回确定性输出        │
  │  示例：fileTreeService, editorService, searchService         │
  └────────────────────┬─────────────────────────────────────────┘
                       │ 调用
  ┌────────────────────▼─────────────────────────────────────────┐
  │  数 据 访 问 层  (repositories/)                             │
  │  Repository 模式：接口 + Supabase 实现，封装所有数据存取      │
  │  上层不直接 import data/ 或 supabase client                   │
  │  示例：fileTreeRepo, trashRepo, supabaseAuthRepo, userRepo   │
  └────────────────────┬─────────────────────────────────────────┘
                       │ 调用 supabase.from() 或 data/
  ┌────────────────────▼─────────────────────────────────────────┐
  │  数 据 层  (Supabase PostgreSQL)                             │
  │  public.file_tree — 树结构（邻接表，ON DELETE CASCADE）       │
  │  public.documents — 文档内容（file_id FK 1:1，JSONB）         │
  │  public.trash     — 回收站（original_id 关联，30 天过期）     │
  │  public.profiles  — 用户画像（id = auth.users.id）            │
  │  认证：auth.users（Supabase Auth 管理）                        │
  └──────────────────────────────────────────────────────────────┘
        ═══════════════════════════════════════════════════════
        被所有层引用

  ┌──────────────────────────────────────────────────────────────┐
  │  基 础 设 施 层  (constants/, utils/, lib/utils.ts)          │
  │  通用工具：ID 生成、日期格式化、cn()、校验正则、常量           │
  └──────────────────────────────────────────────────────────────┘
  ┌──────────────────────────────────────────────────────────────┐
  │  类 型 定 义 层  (types/)                                    │
  │  TypeScript 接口、类型别名，零依赖，被所有层引用               │
  └──────────────────────────────────────────────────────────────┘
```

### 依赖规则

| 规则 | 说明 |
|------|------|
| **依赖单向** | 上层调用下层，下层绝不依赖上层 |
| **禁止跨层** | UI 层不能直接调用业务逻辑层或数据层 |
| **纯函数优先** | 业务逻辑层全部为纯函数，给定输入必定返回确定性输出 |
| **类型安全** | 全量 TypeScript，禁止 `any` |
| **组件小而专** | 每个组件只做一件事，超过 200 行即考虑拆分 |
| **Repository 封装** | 数据访问统一通过 `@/repositories`，不直接 import `@/data/` |
| **路径别名** | 跨目录导入必须使用 `@/` 别名，禁止深层相对路径 |
| **UI 零副作用** | 禁止 `window.prompt`/`confirm`，统一通过 `uiStore.openConfirm()` + `ConfirmDialog` |

### 各页面架构

#### `/workspace` 主页

```
page.tsx
  import { useFileTreeStore } from "@/store/fileTreeStore"
  import { useEditorStore } from "@/store/editorStore"
  import { EDITOR_MENU_ACTIONS } from "@/constants/workspace"
  import { useFolderItems, useBreadcrumb, ... } from "@/hooks/use-file-tree"

  <SidebarLayout>                           # 不再传递 userInfo prop
    ├── <SidebarContent>                    # 拆分出的自包含侧边栏组件
    │     ├── <UserSection />               # useAuth().user → 真实用户信息
    │     ├── <SearchBar />                 # 搜索输入
    │     ├── <FileTree /> / <SearchResultsView />
    │     └── <SidebarFooter />
    ├── 折叠时: <CollapsedTopBar />          # 拆分出的折叠态顶部栏
    │
    ├── currentView === "editor" ?
    │     <DocumentEditor menuActions={EDITOR_MENU_ACTIONS}>
    │       <EditorHeader />               # 面包屑 + 保存状态 + 下拉菜单
    │       <EditorBody />                 # TipTap 富文本编辑器
    │     />
    │   : <FolderView>
    │       <Breadcrumb />
    │       <FolderListItem />             # ★ 内联 <input> 重命名（无 window.prompt）
    │     />
    </SidebarLayout>

  数据流:
    UI 选择节点 → store.selectNode() → 自动切换视图
    UI 编辑器 → editorStore.loadEditorFromTree(id) → findNode() → setContent()
    创建文件 → store.createFile() → fileTreeRepo.insertNode() → addChildToTree()
    重命名   → 内联 <input> 编辑 → store.commitRename() → fileTreeRepo.updateNode()
    删除     → uiStore.openConfirm() → store.deleteNode() → fileTreeRepo.moveToTrash()
    保存     → editorStore.performSave() → editorService.saveDocument() → repo
    用户信息 → UserSection 内部 useAuth().user，无需外部传参
```

#### `/trash` 回收站

```
page.tsx
  import { useTrashStore } from "@/store/trashStore"

  <SidebarLayout isTrashActive>
    └── <TrashHeader />
        <TrashItem />
        <FloatingBatchBar />
    </SidebarLayout>

  数据流:
    恢复 → trashStore.restoreItem() → trashRepo.restoreItem()
           → trashStore 内部自动 loadTree()（TrashPage 不再手动调用）
    彻底删除 → trashStore.deletePermanently() → trashRepo.removeItem()
    批量操作 → batchRestore / batchDelete
    跳转   → router.push('/workspace')（仅 UI 层）
```

### 5.1 `fileTreeStore.ts`

| 状态 | 类型 | 说明 |
|------|------|------|
| `_tree` | `TreeNode[]` | 完整文件树（永不空） |
| `selectedNodeId` | `string \| null` | 当前选中节点 |
| `expandedIds` | `Set<string>` | 已展开的文件夹 ID |
| `currentView` | `"editor" \| "folder"` | 当前视图 |
| `creatingNodeId` | `string \| null` | 正在新建的节点 |
| `loading` | `boolean` | 加载中 |
| `error` | `string \| null` | 错误信息 |

| Action | 说明 |
|--------|------|
| `loadTree()` | 从 Supabase 拉取树，空时注入 `getDefaultRoot()` |
| `selectNode(id)` | 选中节点，自动切换视图 |
| `toggleExpand(id)` | 切换文件夹展开/折叠 |
| `createFile(parentId)` | 创建文件 → `fileTreeRepo.insertNode()` → `addChildToTree()` |
| `createFolder(parentId)` | 创建文件夹 |
| `validateCreateName(nodeId, name)` | 校验名称合法性 |
| `commitRename(nodeId, name)` | 乐观重命名 → `fileTreeRepo.updateNode()` |
| `cancelCreate(nodeId)` | 取消新建 |
| `deleteNode(nodeId)` | 移动到回收站 → `collectDescendantIds()` → `fileTreeRepo.moveToTrash()` |
| `getTree()` | ★ 公开 selector：获取当前树 |
| `getSelectedNode()` | ★ 公开 selector：获取当前选中节点 |

### 5.2 `editorStore.ts`

| 状态 | 类型 | 说明 |
|------|------|------|
| `content` | `DocNode` | TipTap 文档 JSON |
| `title` | `string` | 文档标题 |
| `wordCount` | `number` | 字数 |
| `isSaved` | `boolean` | 是否已保存 |
| `isDirty` | `boolean` | 是否有未保存变更 |

| Action | 说明 |
|--------|------|
| `setContent(doc)` | 更新内容 → 2s 防抖自动保存 |
| `setTitle(title)` | 更新标题 → 2s 防抖自动保存 |
| `performSave()` | ★ 内部调 `editorService.saveDocument({ nodeId, title, content, wordCount })` |

## 十一、完整数据流示例

### 创建文件

```
TreeItemMenu (UI) 点击 [+→新建文档]
  → useFileTreeStore().createFile(parentId)
    → supabaseAuthRepo.getSession() → userId
    → fileTreeRepo.insertNode({ user_id, parent_id, name, type, level, sort_order })
      → supabase.from('file_tree').insert() → 返回 UUID
    → addChildToTree(localTree, parentId, newNode)  # 本地更新
    → set({ _tree: updated, selectedNodeId, creatingNodeId })
```

## 十二、禁止事项

| ❌ 禁止 | ✅ 应该 |
|--------|--------|
| 在 UI 组件中使用 `window.prompt`/`window.confirm` | 内联 `<input>` 重命名 + `uiStore.openConfirm()` |
| 在 UI 组件中写业务规则 | 调用 store actions 或 service 纯函数 |
| Store 跳过 Service 直接操作 Repository（如直接用 `fileTreeRepo.updateNode`） | 通过 `editorService.saveDocument()` 等编排 |
| UI/Store/Service 直接 `import from "@/data/"` | 统一 `import { xxxRepo } from "@/repositories"` |
| 使用 `any` 类型 | 使用 `types/` 中定义的接口 |
| 反向依赖（下层 import 上层） | 保持依赖单向从上到下 |
| 页面组件手动传 mock 用户数据 | `UserSection` 内部 `useAuth().user` |
| `hooks/use-file-tree.ts` 直接访问 `s._tree` | 使用 `getTree()` / `getSelectedNode()` 公开 selector |

## 十三、开发顺序

在添加新功能时，按以下顺序生成代码：

1. **types/** — 定义接口
2. **constants/** + **utils/** — 定义常量、工具函数、校验器
3. **repositories/** — 接口 + 实现 + 注册实例
4. **lib/services/** — 实现业务逻辑纯函数
5. **store/** — 实现 Zustand store（调用 services + repositories）
6. **hooks/** — 实现共享状态 hook（如需要）
7. **components/** — 实现 UI 组件（从 store/hooks 读取数据和调用 actions）
8. **app/** — 组合页面
