> 更新时间：2026-06-01

## 一、六层架构总览

```
┌──────────────────────────────────────────────────────────┐
│  ＵＩ 层  (components/ + app/)                           │
│  只负责渲染和事件派发，不包含业务逻辑、校验、数据存取       │
│  示例：FileTree.tsx, DocumentEditor.tsx                  │
└────────────────────┬─────────────────────────────────────┘
                     │ 调用
┌────────────────────▼─────────────────────────────────────┐
│  状 态 层  (hooks/ + lib/auth-context.tsx)                │
│  管理全局/局部状态，提供 actions 供 UI 层调用              │
│  actions 内部调用业务逻辑层，不直接操作 DOM 或 mock 数据    │
│  示例：use-workspace.ts, use-sidebar.ts                  │
└────────────────────┬─────────────────────────────────────┘
                     │ 调用
┌────────────────────▼─────────────────────────────────────┐
│  业 务 逻 辑 层  (lib/services/)                         │
│  纯函数，实现所有业务规则：校验、去重、生成、树操作          │
│  不依赖 React、状态管理、DOM，给定输入必定返回确定性输出     │
│  示例：tree-service.ts                                  │
└────────────────────┬─────────────────────────────────────┘
                     │ 调用
┌────────────────────▼─────────────────────────────────────┐
│  数 据 层  (data/)                                       │
│  Mock 数据和简单存取函数，未来可替换为 localStorage / API   │
│  示例：workspace-mock.ts, auth-mock.ts, users-mock.ts    │
└────────────────────┬─────────────────────────────────────┘
                     │ 使用
┌────────────────────▼─────────────────────────────────────┐
│  基 础 设 施 层  (constants/, lib/utils.ts, lib/validators.ts) │
│  通用工具：ID 生成、日期格式化、cn()、校验正则、常量        │
│  被所有层调用，不调用任何其他层                            │
└────────────────────┬─────────────────────────────────────┘
                     │ 使用
┌────────────────────▼─────────────────────────────────────┐
│  类 型 定 义 层  (types/)                                │
│  TypeScript 接口、类型别名，零依赖，被所有层引用            │
│  示例：workspace.ts                                     │
└──────────────────────────────────────────────────────────┘
```

### 依赖规则

| 规则 | 说明 |
|------|------|
| **依赖单向** | 上层调用下层，下层绝不依赖上层 |
| **禁止跨层** | UI 层不能直接调用业务逻辑层或数据层 |
| **纯函数优先** | 业务逻辑层全部为纯函数，给定输入必定返回确定性输出 |
| **类型安全** | 全量 TypeScript，禁止 `any` |
| **组件小而专** | 每个组件只做一件事，超过 200 行即考虑拆分 |

### 核心 Hook 数据流

```
use-workspace.ts
├── useState<TreeNode[]>(initialTree)          → 文件树数据
├── useState<string | null>(selectedId)        → 当前选中节点
├── useState<"editor" | "folder">(currentView) → 当前视图
├── useState<string | null>(creatingNodeId)    → 新建中的节点
│
├── actions 调用 lib/services/tree-service.ts
│   ├── findNode()        → 查找节点
│   ├── removeNode()       → 删除节点
│   ├── getSiblingNames()  → 获取同级名称
│   ├── validateName()     → 名称校验
│   ├── generateUniqueName() → 生成唯一名称
│   └── buildBreadcrumb()  → 构建面包屑
│
└── 派生值（folderItems, breadcrumb, documentInfo）
```

### tree-service.ts 导出函数

| 函数 | 类型 | 职责 |
|------|------|------|
| `generateId()` | `() => string` | 生成唯一节点 ID |
| `formatTime()` | `() => string` | 格式化当前时间 |
| `findNode(nodes, id)` | `(TreeNode[], string) => TreeNode \| null` | 深度查找节点 |
| `removeNode(nodes, id)` | `(TreeNode[], string) => boolean` | 删除节点（原地修改） |
| `getParent(nodes, childId)` | `(TreeNode[], string) => TreeNode \| null` | 查找父节点 |
| `getSiblingNames(nodes, nodeId)` | `(TreeNode[], string) => string[]` | 获取同级节点名称列表 |
| `validateName(name, siblingNames)` | `(string, string[]) => string \| null` | 校验名称合法性 |
| `generateUniqueName(baseName, existingNames)` | `(string, string[]) => string` | 生成不重复名称 |
| `buildBreadcrumb(tree, targetId)` | `(TreeNode[], string) => BreadcrumbItem[]` | 构建面包屑路径 |

## 十、禁止事项

| ❌ 禁止 | ✅ 应该 |
|---------|--------|
| 在 UI 组件中写业务规则 | 调用 hooks 中的 actions |
| 在 hooks 中直接读写 mock 数据源 | 调用 lib/services 中的纯函数 |
| 跳过业务逻辑层直接操作数据 | 严格按 UI → 状态 → 业务逻辑 → 数据 |
| 使用 `any` 类型 | 使用 `types/workspace.ts` 中定义的接口 |
| 将 mock 数据定义在页面或组件内部 | 统一定义在 `data/` 中 |
| 反向依赖（下层 import 上层） | 保持依赖单向从上到下 |

## 十一、开发顺序

在添加新功能时，按以下顺序生成代码：

1. **types/** — 定义接口
2. **constants/** — 定义常量和工具函数
3. **data/** — 创建 mock 数据和存取函数
4. **lib/services/** — 实现业务逻辑纯函数
5. **hooks/** — 实现状态管理（调用 services）
6. **components/** — 实现 UI 组件（从 hooks 读取数据和调用 actions）
