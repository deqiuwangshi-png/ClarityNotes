> 本文档记录 ClarityNotes（心智笔记）的八层前端架构与项目目录结构。
> 更新时间：2026-06-02

## 一、八层架构总览

```
┌──────────────────────────────────────────────────────────────────┐
│  ＵＩ 层  (components/ + app/)                                   │
│  只负责渲染和事件派发，不包含业务逻辑、校验、数据存取              │
│  示例：FolderView.tsx, AccountTab.tsx, DocumentEditor.tsx        │
└────────────────────┬─────────────────────────────────────────────┘
                     │ 调用 useAuth() / useFileTreeStore() / useSidebar()
┌────────────────────▼─────────────────────────────────────────────┐
│  状 态 层  (store/ + lib/auth-context.tsx + hooks/)              │
│  Zustand stores 管理全局状态，auth-context 管理认证状态           │
│  hooks/ 提供可复用的局部/共享状态逻辑                              │
│  状态层调用 service 和 repository，不直接操作 DOM 或原始数据       │
│  示例：fileTreeStore.ts, trashStore.ts, use-sidebar.ts           │
└────────────────────┬─────────────────────────────────────────────┘
                     │ 调用
┌────────────────────▼─────────────────────────────────────────────┐
│  业 务 逻 辑 层  (lib/services/)                                 │
│  纯函数，实现所有业务规则：校验、去重、生成、树操作、权限判断       │
│  不依赖 React、状态管理、DOM，给定输入必定返回确定性输出            │
│  示例：fileTreeService.ts, auth-service.ts, userService.ts       │
└────────────────────┬─────────────────────────────────────────────┘
                     │ 调用
┌────────────────────▼─────────────────────────────────────────────┐
│  数 据 访 问 层  (repositories/)                                 │
│  Repository 模式：接口 + Mock 实现，封装所有数据存取              │
│  上层不直接 import data/，通过 repository 实例统一访问            │
│  切换真实 API 只需改 index.ts 中的实例                            │
│  示例：userRepo, fileTreeRepo, trashRepo, workspaceRepo, sessionRepo │
└────────────────────┬─────────────────────────────────────────────┘
                     │ 调用
┌────────────────────▼─────────────────────────────────────────────┐
│  数 据 层  (data/)                                               │
│  Mock 初始数据 + 内存状态的存取函数（仅被 repositories/ 调用）     │
│  示例：mockUsers.ts, mockFileTree.ts, mockTrash.ts              │
└────────────────────┬─────────────────────────────────────────────┘
                     │ 使用
┌────────────────────▼─────────────────────────────────────────────┐
│  基 础 设 施 层  (constants/, utils/, lib/utils.ts)              │
│  通用工具：ID 生成、日期格式化、cn()、校验正则、常量               │
│  被所有层调用，不调用任何其他层                                    │
└────────────────────┬─────────────────────────────────────────────┘
                     │ 使用
┌────────────────────▼─────────────────────────────────────────────┐
│  类 型 定 义 层  (types/)                                        │
│  TypeScript 接口、类型别名，零依赖，被所有层引用                    │
│  示例：auth.ts, fileTree.ts, workspace.ts, landing.ts            │
└──────────────────────────────────────────────────────────────────┘
```

### 依赖规则

| 规则 | 说明 |
|------|------|
| **依赖单向** | 上层调用下层，下层绝不依赖上层 |
| **禁止跨层** | UI 层不能直接调用业务逻辑层或数据层 |
| **纯函数优先** | 业务逻辑层全部为纯函数，给定输入必定返回确定性输出 |
| **类型安全** | 全量 TypeScript，禁止 `any` |
| **组件小而专** | 每个组件只做一件事，超过 200 行即考虑拆分 |
| **Repository 封装** | 数据访问统一通过 `@/repositories`，不直接 import `@/data/` |
| **路径别名** | 跨目录导入必须使用 `@/` 别名，禁止深层相对路径 |

### 各页面架构流程

#### `/workspace` 主页

```
page.tsx
  ├── import { fileTreeStore, useBreadcrumb, useFolderItems, ... } from "@/store/fileTreeStore"
  ├── import { workspaceRepo } from "@/repositories"
  │
  ├── <SidebarLayout userInfo={workspaceRepo.mockUser}>
  │     ├── <UserSection /> → 用户头像 + 折叠按钮 + 设置菜单
  │     ├── <SearchBar />    → 搜索输入
  │     ├── <FileTree />     → 文件树（递归 TreeNode）
  │     │     └── <TreeItem /> → inline rename + <TreeItemMenu />
  │     ├── <SidebarFooter /> → 垃圾箱 & 精选入口
  │     └── 折叠时顶部栏: <TopBarBreadcrumb /> + ☰ 展开按钮
  │
  │     ├── currentView === "editor" ?
  │     │     <DocumentEditor
  │     │       <EditorHeader /> → 面包屑 + 保存状态 + 下拉菜单
  │     │       <EditorBody />   → TipTap 富文本编辑器
  │     │       <BubbleMenu />   → 选中文本浮动工具栏
  │     │     />
  │     │   : <FolderView>
  │     │       <Breadcrumb />       → 面包屑导航（折叠时隐藏，由 TopBarBreadcrumb 替代）
  │     │       <FolderListHeader /> → 表头
  │     │       <FolderListItem />   → 列表行 + FloatingMenu（重命名/删除）
  │     │     />
  │     </SidebarLayout>
  │
  └── 数据流:
        UI 选择节点 → store.selectNode() → 自动切换视图
        UI 编辑器 → editorStore.loadEditorFromTree(id) → findNode() → setContent()
        UI 点击 → store.action() → service 纯函数 → repo.setTree() → data/
        UI 重命名 → window.prompt() → store.commitRename() → repo.setTree()
        UI 删除   → window.confirm() → store.deleteNode() → repo + trashRepo
```

#### `/trash` 回收站

```
page.tsx
  ├── import { useTrashStore } from "@/store/trashStore"
  ├── import { workspaceRepo } from "@/repositories"
  │
  ├── <SidebarLayout userInfo={workspaceRepo.mockUser} isTrashActive>
  │     └── <TrashHeader />           → 标题 + 清空按钮
  │         <TrashItem />             → 列表行 + FloatingMenu（多选/恢复/彻底删除）
  │         <FloatingBatchBar />      → 批量操作浮动栏（批量恢复/批量删除）
  │     </SidebarLayout>
  │
  └── 数据流:
        恢复 → useTrashStore.restoreItem() → trashService.restoreItem() → repo
        彻底删除 → useTrashStore.deletePermanently() → trashRepo.removeFromTrash()
```

#### `/login` 认证页

```
page.tsx
  ├── import { useAuth } from "@/lib/auth-context"
  │
  ├── <AuthPage>
  │     <AuthCard>
  │       <TabSwitcher />             → 登录/注册标签切换
  │       <AuthForm />                → 统一认证表单
  │       <SocialLogin />             → Google/Apple 社交登录按钮
  │     </AuthCard>
  │   </AuthPage>
  │
  └── 数据流:
        登录 → useAuth().login(email, password)
              → auth-service.validateLogin()
                → userRepo.MOCK_USERS.find(...)
              → sessionRepo.createSession(user) → localStorage
              → setUser(user)
        注册 → useAuth().register(payload)
              → auth-service.validateRegister()
                → userRepo.findUserByEmail() → userRepo.addUser()
              → sessionRepo.createSession(user) → localStorage
              → setUser(user)
        退出 → useAuth().logout()
              → sessionRepo.clearSession()
              → setUser(null)
        注销 → useAuth().logout(true)
              → sessionRepo.clearAllData() → localStorage.clear()
              → setUser(null)
```

## 五、`store/` — 状 态 层（Zustand）

项目使用 **Zustand** 管理全局状态。所有 store 位于 `store/` 目录。

```
store/
├── fileTreeStore.ts                  # ★ 文件树核心状态
├── trashStore.ts                     # 垃圾箱状态
└── editorStore.ts                    # 编辑器状态（自动保存）
```

### 5.1 `fileTreeStore.ts` — 文件树状态

| 状态 | 类型 | 说明 |
|------|------|------|
| `tree` | `TreeNode[]` | 完整文件树 |
| `selectedNodeId` | `string \| null` | 当前选中节点 |
| `expandedIds` | `Set<string>` | 已展开的文件夹 ID |
| `currentView` | `"editor" \| "folder"` | 当前视图 |
| `creatingNodeId` | `string \| null` | 正在新建的节点 |

| Action | 说明 |
|------|------|
| `selectNode(id)` | 选中节点，自动切换视图 |
| `toggleExpand(id)` | 切换文件夹展开/折叠 |
| `createFile(parentId)` | 在文件夹下创建文件 |
| `createFolder(parentId)` | 在文件夹下创建文件夹 |
| `commitRename(nodeId, name)` | 确认重命名 → `fileTreeRepo.setTree()` |
| `cancelCreate(nodeId)` | 取消新建 |
| `deleteNode(nodeId)` | 删除节点 → `trashRepo.addToTrash()` + `fileTreeRepo.setTree()` |
| `setTree(tree)` | 覆盖整个树 |
| `restoreFromTrash(nodeId, cb)` | 从垃圾箱恢复 |

**派生 hooks**：

| Hook | 返回类型 | 说明 |
|------|----------|------|
| `useBreadcrumb()` | `BreadcrumbItem[]` | 当前选中节点的面包屑路径 |
| `useFolderItems()` | `FolderItem[]` | 当前文件夹内的子项列表 |
| `useFolderName()` | `string` | 当前文件夹名称（默认"我的文档"） |
| `useDocumentInfo()` | `DocumentInfo \| null` | 当前文档信息（用于编辑器） |
| `useSelectedNode()` | `TreeNode \| null` | 当前选中的树节点 |

### 5.2 `trashStore.ts` — 垃圾箱状态

| 状态 | 类型 | 说明 |
|------|------|------|
| `items` | `TrashItemData[]` | 垃圾箱列表 |
| `isBatchMode` | `boolean` | 批量选择模式 |
| `selectedIds` | `Set<string>` | 已选中的 ID |

| Action | 说明 |
|------|------|
| `refresh()` | 从 `trashRepo.getTrash()` 刷新 |
| `enterBatchMode(id?)` | 进入批量模式 |
| `exitBatchMode()` | 退出批量模式 |
| `toggleSelectItem(id)` | 切换选中 |
| `toggleSelectAll()` | 全选/取消全选 |
| `restoreItem(id, tree)` | 恢复 → `trashService.restoreItem()` |
| `deletePermanently(id)` | 彻底删除 → `trashRepo.removeFromTrash()` |
| `emptyTrash()` | 清空 → `trashRepo.clearTrash()` |
| `batchRestore(tree)` | 批量恢复 |
| `batchDelete()` | 批量删除 |

### 5.3 `editorStore.ts` — 编辑器状态

| 状态 | 类型 | 说明 |
|------|------|------|
| `content` | `string` | 编辑器 HTML 内容 |
| `title` | `string` | 文档标题 |
| `wordCount` | `number` | 字数统计 |
| `isSaved` | `boolean` | 是否已保存 |
| `isDirty` | `boolean` | 是否有未保存修改 |

| Action | 说明 |
|------|------|
| `loadFromNode(id, content?, title?)` | 从文件树节点加载 |
| `setContent(html)` | 更新内容（触发 2s 自动保存） |
| `setTitle(title)` | 更新标题（触发 2s 自动保存） |
| `saveNow()` | 手动保存 |
| `cancelAutoSave()` | 取消自动保存定时器 |

### 7.1 `auth-context.tsx` — 认证状态

| 导出 | 说明 |
|------|------|
| `AuthProvider` | 包裹根布局，提供认证上下文 |

|------|------|

## 八、`lib/services/` — 业 务 逻 辑 层

```

├── fileTreeService.ts                # ★ 文件树纯函数集合

### 8.1 `auth-service.ts`

| 函数 | 说明 |

## 十四、完整数据流示例

### 创建文件

```
TreeItemMenu (UI) 点击 [+→新建文档]
  → useFileTreeStore().createFile(parentId)
    → fileTreeService.createNode(tree, parentId, "file")
      → findNode() 找父节点
      → validateName() 生成唯一名称
      → generateId() 生成新 ID
      → formatTimestamp() 生成时间戳
      → 返回 { newTree, createdNode }
    → fileTreeRepo.setTree(newTree)
      → data/mockFileTree.setTree(newTree)
    → set({ tree: newTree, selectedNodeId, currentView: "editor" })
```

### 重命名文件

```
FolderListItem (UI) 点击 [⋮ → 重命名]
  → window.prompt("请输入新名称")
  → useFileTreeStore().commitRename(id, name)
    → fileTreeService.renameNode(tree, id, name)
      → getSiblingNames() 取同级名称
      → validateName() 校验冲突
      → 返回 { newTree }
    → fileTreeRepo.setTree(newTree)
    → set({ tree: newTree })
```

### 删除文件到垃圾箱

```
TreeItemMenu (UI) 点击 [⋮ → 删除]
  → useFileTreeStore().deleteNode(id)
    → fileTreeService.moveToTrash(tree, id)
      → deleteNode() 删除并返回原节点
      → 生成 TrashItemData
    → trashRepo.addToTrash(trashItem)
    → fileTreeRepo.setTree(newTree)
    → set({ tree: newTree })
```

### 修改个人信息

```

  → validateFullName(name)  ← utils/validators.ts

      → userRepo.updateUserInStorage(updated)

    → userService.clearAllUserData()

## 十五、禁止事项

| ❌ 禁止 | ✅ 应该 |
|---------|--------|
| 在 UI 组件中写业务规则 | 调用 store actions 或 service 纯函数 |
| 在 UI 组件中直接读写 localStorage | 通过 `updateUser()` 或 Repository |
| UI/Store/Service 直接 `import from "@/data/"` | 统一 `import { xxxRepo } from "@/repositories"` |
| 在 store 中直接操作原始数据 | 调用 service 纯函数 + repo 存取 |
| 跳过业务逻辑层直接操作数据 | 严格按 UI → Store → Service → Repository → Data |
| 使用 `any` 类型 | 使用 `types/` 中定义的接口 |
| 将 mock 数据定义在页面或组件内部 | 统一定义在 `data/` 中 |
| 反向依赖（下层 import 上层） | 保持依赖单向从上到下 |
| 跨层深依赖（UI→Service→Data 跳过 Repository） | 每层只能调用紧邻下层 |

## 十六、开发顺序

在添加新功能时，按以下顺序生成代码：

1. **types/** — 定义接口
2. **constants/** + **utils/** — 定义常量、工具函数、校验器
3. **data/** — 创建/修改 mock 数据和存取函数
4. **repositories/** — 接口 + Mock 实现 + 注册实例
5. **lib/services/** — 实现业务逻辑纯函数
6. **store/** — 实现 Zustand store（调用 services + repositories）
7. **hooks/** — 实现共享状态 hook（如需要）
8. **components/** — 实现 UI 组件（从 store/hooks 读取数据和调用 actions）
