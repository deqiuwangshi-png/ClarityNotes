# 应用主页 - React 组件拆分方案

> **技术栈**: Next.js 16 (App Router) + React 19 + TypeScript + Tailwind CSS 4
> **路径别名**: `@/*` 映射至项目根目录
> **设计系统**: 基于 `mint-*` 配色体系（`--mint-bg`, `--mint-text`, `--mint-accent` 等），使用 Tailwind 自定义主题色 + Material Symbols 图标库。

***

## 一、整体页面架构

### 1.1 页面级组件

| 组件名             | 文件路径                              | 类型    | 核心职责                                                                      |
| --------------- | --------------------------------- | ----- | ------------------------------------------------------------------------- |
| `WorkspacePage` | `@/app/(main)/workspace/page.tsx` | 服务端页面 | 作为应用主页的入口页面组件，负责组装 `AppShell`，管理侧边栏折叠状态、当前视图切换（编辑器 / 文件夹浏览器）、文档选中状态的顶层状态。 |

### 1.2 组件层级树

```
WorkspacePage                          // 页面入口 - 管理全局 UI 状态
├── AppShell                           // 布局容器 - flex 横向布局
│   ├── Sidebar                        // 侧边栏容器
│   │   ├── UserProfile                // 用户信息 + 折叠按钮
│   │   ├── SearchBar                  // 搜索框 + 新建笔记按钮
│   │   ├── FileTree                   // 文件树导航容器
│   │   │   └── FileTreeItem[]         // 递归树节点（文件夹 / 文件）
│   │   └── SidebarFooter              // 底栏：精选 / 回收站入口
│   ├── SidebarToggle                  // 侧边栏折叠后的展开按钮（浮动）
│   └── MainContent                    // 右侧主内容区容器
│       ├── EditorPanel                // 文档编辑视图
│       │   ├── EditorHeader           // 编辑器顶栏
│       │   │   ├── Breadcrumb         // 面包屑导航
│       │   │   ├── SaveStatusBadge    // 自动保存状态指示器
│       │   │   └── TopBarDropdown     // 更多操作下拉菜单
│       │   └── EditorBody             // 编辑器正文区域
│       └── FolderBrowser              // 文件夹浏览视图
│           ├── FolderBreadcrumb       // 文件夹面包屑
│           ├── FolderListHeader       // 列标题栏
│           └── FolderListItem[]       // 文件夹 / 文件条目
└── BubbleMenu                         // 文本选中浮动工具栏（全局层级）
```

### 1.3 数据流概览

```
WorkspacePage (状态所有者)
  │
  ├─ sidebarCollapsed: boolean ──────► Sidebar, SidebarToggle, AppShell
  ├─ currentView: 'editor' | 'folder' ► MainContent → EditorPanel | FolderBrowser
  ├─ selectedFileId: string ─────────► FileTree, EditorBody, Breadcrumb
  ├─ fileTreeData: TreeNode[] ───────► FileTree → FileTreeItem[]
  └─ folderData: FolderItem[] ───────► FolderBrowser → FolderListItem[]

所有子组件通过 Props 接收数据，通过回调函数向上通知事件。
```

## 二、组件详细拆分

### 2.1 AppShell（应用外壳）

| 属性       | 内容                                                                               |
| -------- | -------------------------------------------------------------------------------- |
| **组件名**  | `AppShell`                                                                       |
| **文件路径** | `@/components/workspace/app-shell.tsx`                                           |
| **类型**   | 客户端组件（`'use client'`）                                                            |
| **核心职责** | 提供应用主页的顶级布局容器，采用 `flex h-screen overflow-hidden` 横向放置侧边栏与主内容区。负责管理侧边栏展开/折叠的动画过渡。 |

**包含的 HTML 元素**：

- `<div id="app-shell" class="flex h-screen overflow-hidden">` — 顶级 flex 容器

**子组件插槽**：

- `children`: 接收 `Sidebar` + `SidebarToggle` + `MainContent`

**Props 接口**：

```typescript
interface AppShellProps {
  children: React.ReactNode;
}
```

**状态与行为**：纯布局组件，无独立状态。仅提供 flex 容器和溢出控制。

**样式作用域**：`flex h-screen overflow-hidden` — 全屏 Flex 布局，禁止页面级滚动。

**交互逻辑**：无交互逻辑，仅作为布局容器。

### 2.2 Sidebar（侧边栏）

| 属性       | 内容                                            |
| -------- | --------------------------------------------- |
| **组件名**  | `Sidebar`                                     |
| **文件路径** | `@/components/workspace/sidebar.tsx`          |
| **类型**   | 客户端组件（`'use client'`）                         |
| **核心职责** | 左侧导航面板，承载用户信息、搜索入口、文件树导航、底栏快捷入口。管理自身的折叠/展开动画。 |

**包含的 HTML 元素**：

- `<aside id="sidebar">` — 侧边栏容器，带 `w-[280px]` 宽度、`border-r` 右边框、`transition-all duration-250` 过渡动画、`shadow-sm` 阴影
- 内部 `<div class="flex flex-col h-full p-5">` — 垂直弹性布局内壳

**子组件**：

- `UserProfile`
- `SearchBar`
- `FileTree`
- `SidebarFooter`

**Props 接口**：

```typescript
interface SidebarProps {
  collapsed: boolean;
  onToggle: () => void;
  fileTreeData: TreeNode[];
  selectedFileId: string | null;
  onFileSelect: (fileId: string) => void;
  onFileCreate: (parentId: string) => void;
  onFileMore: (fileId: string) => void;
  onSearch: (query: string) => void;
  onNewNote: () => void;
  userInfo: UserInfo;
}
```

**状态与行为**：

- 通过 `collapsed` prop 控制宽度动画：展开 `w-[280px] opacity-100`，折叠 `w-0 opacity-0 p-0 border-none`
- 过渡使用 `cubic-bezier(0.2, 0.9, 0.4, 1.1)` 弹性曲线
- 内部 `sidebar-scroll` 自定义滚动条样式

**样式作用域**：`w-[280px]`, `bg-mint-bg`, `border-r border-mint-border/30`, `transition-all duration-250`, `shadow-sm`。滚动条宽度 4px、滑块色 `#bdc9c4`。

**交互逻辑**：

- 接收 `collapsed` prop 响应父组件状态变化
- 通过 `onToggle` 回调通知父组件切换侧边栏
- 透传文件操作回调给子组件

### 2.5 FileTree（文件树导航）

| 属性       | 内容                                                              |
| -------- | --------------------------------------------------------------- |
| **组件名**  | `FileTree`                                                      |
| **文件路径** | `@/components/workspace/file-tree.tsx`                          |
| **类型**   | 客户端组件（`'use client'`）                                           |
| **核心职责** | 以树形结构展示用户文档层级（文件夹与文件），支持展开/折叠、选中高亮、悬停操作按钮。使用 ARIA tree 模式实现无障碍。 |

**包含的 HTML 元素**：

- `<nav role="tree" aria-label="文件树">` — 语义化导航容器
- `flex-1 min-h-0 overflow-y-auto pr-1 sidebar-scroll` — 可滚动区域

**子组件**：

- `FileTreeItem`（递归渲染）

**Props 接口**：

```typescript
interface FileTreeProps {
  nodes: TreeNode[];
  selectedId: string | null;
  onSelect: (nodeId: string) => void;
  onToggle: (nodeId: string) => void;
  onCreate: (parentId: string) => void;
  onMore: (nodeId: string) => void;
}

interface TreeNode {
  id: string;
  name: string;
  type: 'folder' | 'file';
  level: number;
  expanded: boolean;
  children?: TreeNode[];
}
```

**状态与行为**：

- 树节点的展开/折叠状态由父组件 `WorkspacePage` 管理，通过 `TreeNode.expanded` 传递
- 点击文件夹箭头触发 `onToggle(nodeId)` 展开/折叠子节点
- 选中节点高亮：`bg-mint-hover font-medium text-mint-accent-light` + 右侧 `w-0.5 rounded-l bg-[#26917A]` 指示条

**样式作用域**：`flex-1 min-h-0 overflow-y-auto pr-1`。滚动区域使用 `sidebar-scroll` 类自定义滚动条。

**交互逻辑**：

- 点击行 → `onSelect(nodeId)`，切换选中状态
- 点击展开箭头 → `onToggle(nodeId)`，展开/折叠子节点
- 悬停行 → 显示操作按钮组（`add` + `more_vert`）
- 选中节点显示绿色左侧指示条

### 2.6 FileTreeItem（文件树节点）

| 属性       | 内容                                                           |
| -------- | ------------------------------------------------------------ |
| **组件名**  | `FileTreeItem`                                               |
| **文件路径** | `@/components/workspace/file-tree-item.tsx`                  |
| **类型**   | 客户端组件（`'use client'`）                                        |
| **核心职责** | 渲染文件树中的单个节点，支持文件夹（可展开/折叠、带子节点）和文件（叶子节点）两种类型。每种类型有不同的 SVG 图标。 |

**包含的 HTML 元素**：

- `<div role="treeitem" aria-level={level} aria-expanded={expanded} aria-selected={selected}>` — 树节点容器，`h-8` 固定行高
- 展开/折叠箭头 `<span>` — 仅文件夹显示，`rotate-0`（展开）或 `-rotate-90`（折叠），内含 `expand_more` 图标
- 文件夹图标（自定义 SVG 三层路径：填充 + 描边）或文件图标（自定义 SVG：文档轮廓 + 内部线条）
- 节点名称 `<span class="truncate">`
- 操作按钮组 — 悬停时显示：`add` 按钮（仅文件夹）、`more_vert` 按钮
- 选中指示条 `<span class="absolute right-0 top-1 h-6 w-0.5 rounded-l bg-[#26917A]">`

**Props 接口**：

```typescript
interface FileTreeItemProps {
  node: TreeNode;
  selected: boolean;
  onSelect: (nodeId: string) => void;
  onToggle: (nodeId: string) => void;
  onCreate: (parentId: string) => void;
  onMore: (nodeId: string) => void;
}
```

**状态与行为**：

- 递归渲染子节点：当 `node.expanded && node.children` 时，渲染嵌套 `FileTreeItem` 列表
- 折叠状态通过 CSS `-rotate-90` 旋转箭头图标表示
- `padding-left` 通过 `style={{ paddingLeft: level * 16 + 8 }}` 动态计算缩进
- 悬停时操作按钮组由 `opacity-0 group-hover/tree-row:opacity-100` 控制显示

**样式作用域**：

- 基础行：`h-8 w-full rounded-md text-[13px] transition hover:bg-mint-hover/70`
- 选中态：`bg-mint-hover font-medium text-mint-accent-light`
- 未选中态：`text-[#374151]`
- 文件夹图标 SVG 选中态使用 `#007A66` / `#E4F5EF`，未选中使用 `#6B7280` / `#FFFFFF`
- 文件图标 SVG 选中态使用 `#007A66` / `#E4F5EF`，未选中使用 `#6B7280` / `#FFFFFF`
- 选中指示条：`w-0.5 bg-[#26917A]`

**交互逻辑**：

- 点击行本体 → `onSelect(node.id)`
- 点击展开箭头 → `onToggle(node.id)`
- 点击 `add` 按钮 → `onCreate(node.id)`（仅在文件夹节点）
- 点击 `more_vert` 按钮 → `onMore(node.id)`

## 八、Next.js 特定实现注意事项

1. **客户端组件边界**：`'use client'` 指令仅标注在包含交互逻辑（`useState`, `useEffect`, 事件处理）的组件上。纯展示组件（`SaveStatusBadge`、`FolderListHeader`）保留为服务端组件。
2. **图标方案**：Material Symbols 通过 `<link>` 在 `layout.tsx` 中全局引入，组件内使用 `<span className="material-symbols-outlined">icon_name</span>` 渲染。后续可封装为 `<Icon name="search" filled={false} />` 组件统一管理。
3. **Tailwind 4 主题色**：项目 `globals.css` 中已使用 `@theme inline` 定义 CSS 变量。需在 Tailwind 配置中扩展 `mint-*` 色值，或将 `mint-*` 色值定义为 CSS 自定义属性后通过任意值语法（如 `bg-[--mint-bg]`）使用。推荐做法是在 `globals.css` 中添加 `@theme inline` 块定义 `--color-mint-*`。
4. **路径别名**：所有跨目录导入必须使用 `@/` 别名（如 `import { Sidebar } from '@/components/workspace/sidebar'`），同目录下可使用相对路径。
5. **CSS 动画**：Sidebar 折叠/展开动画由 Tailwind `transition-all duration-250` + `cubic-bezier(0.2, 0.9, 0.4, 1.1)` 实现。BubbleMenu 出现/消失动画通过 CSS `transition` + `scale/translate` 实现。建议封装自定义 CSS 类或使用 Tailwind `@apply` 指令。
6. **ARIA 无障碍**：`FileTree` 使用 `role="tree"`、`role="treeitem"`、`aria-level`、`aria-expanded`、`aria-selected` 属性。所有图标按钮均配备 `aria-label`。
