# 认证模块（登录/注册）- React 组件拆分方案

> **技术栈**: Next.js 16 (App Router) + React 19 + TypeScript + Tailwind CSS 4
> **路径别名**: `@/*` 映射至项目根目录
> **设计系统**: 基于 "Mint Outline" 配色体系的子集，独立于工作区和官网页面的认证专用令牌集（`surface-container: #ffffff`、`secondary-container: #f1f5f1`、字号体系：h1/h2/body-md(14px)/label-sm），Inter 字体排版，Material Symbols 图标库，卡片居中布局。

---

## 一、设计令牌对比声明

> **重要**: 认证页使用**独立的令牌值**，与官网（`03官网组件.md`）和工作区均不同。下表列出认证页与现有 `globals.css` 的差异，实现时必须按认证页的 HTML 原型覆盖。

| 令牌 | 认证页 (认证.html) | 现有 globals.css | 差异说明 |
|------|--------------------|-------------------|----------|
| `surface-container` | `#ffffff` | `#ebefec` | 认证页使用纯白卡片背景 |
| `secondary-container` | `#f1f5f1` | `#aeedd5` | 完全不同的色调，用于标签切换栏底色 |
| `on-surface-variant` | `#3e4945` | `#3e4946` | 认证页使用 `#3e4945`（绿色调），官网使用 `#3e4946`（灰色调） |
| `body-md` 字号 | `14px` / line-height `1.5` / w `400` | `16px` / line-height `1.5` / w `400` | 认证页字号为 14px 而非 16px |
| `label-sm` 字号 | `12px` / line-height `1` / letter-spacing `0.02em` / w `500` | 不存在 `label-sm` | 认证页独有标签字号（官网使用 `label-caps: 12px` w600） |
| `h1` 字号 | `32px` / line-height `1.2` / letter-spacing `-0.02em` / w `700` | 不存在 `32px` 语义令牌 | 认证页独有的大标题字号 |
| `h2` 字号 | `24px` / line-height `1.3` / letter-spacing `-0.01em` / w `600` | 官网有 `headline-md: 24px / 1.3 / 600` | 语义名不同，但值相同 |

### 1.1 认证页完整设计令牌清单

```css
/* 颜色令牌（以认证.html tailwind.config 为准） */
--color-primary: #006b5b;
--color-on-primary: #ffffff;
--color-surface: #f6faf7;
--color-on-surface: #181d1b;
--color-surface-container: #ffffff;
--color-outline: #6e7a75;
--color-outline-variant: #bdc9c4;
--color-on-surface-variant: #3e4945;
--color-secondary-container: #f1f5f1;

/* 字号令牌（与官网体系不同，需独立定义） */
--font-size-h1: 32px;        /* line-height: 1.2, letter-spacing: -0.02em, font-weight: 700 */
--font-size-h2: 24px;        /* line-height: 1.3, letter-spacing: -0.01em, font-weight: 600 */
--font-size-body-md: 14px;   /* line-height: 1.5, font-weight: 400 */
--font-size-label-sm: 12px;  /* line-height: 1, letter-spacing: 0.02em, font-weight: 500 */

/* 圆角令牌 */
--radius-xl: 0.75rem;    /* 12px — input / tab 容器级别 */
--radius-2xl: 1.5rem;    /* 24px — 卡片容器级别 */
--radius-lg: 0.5rem;     /* 8px — 激活标签级别 */
```

## 二、整体页面架构

### 2.1 页面级组件

| 组件名 | 文件路径 | 类型 | 核心职责 |
|--------|----------|------|---------|
| `AuthPage` | `@/app/(auth)/login/page.tsx` 或 `@/app/auth/page.tsx` | 客户端页面 | 作为登录/注册入口，组合认证卡片中的所有子组件。管理登录/注册表单切换状态、表单提交流程。 |

### 2.2 组件层级树

```
AuthPage                                    // 页面入口 - 管理 tab 切换状态与表单提交
└── AuthCard                                // 白色圆角卡片容器（居中布局）
    ├── TabSwitcher                         // 登录/注册切换标签栏
    │   ├── TabButton (登录)                // "登录" 标签按钮（激活态 bg-primary text-on-primary）
    │   └── TabButton (注册)                // "注册" 标签按钮（非激活态 text-on-surface-variant）
    ├── LoginForm                           // 登录表单（条件渲染）
    │   ├── FormTitle                       // "欢迎回来" 标题
    │   ├── FormField（邮箱）               // 邮箱输入字段（label + input）
    │   ├── FormField（密码）               // 密码输入字段 + "忘记密码？"链接
    │   ├── AuthCheckbox                    // "记住我" 复选框
    │   └── SubmitButton                    // "立即登录" 提交按钮（含 loading 态）
    ├── SocialLoginDivider                  // "或者通过以下方式" 分割线
    ├── SocialLoginButtons                  // Google + Apple 第三方登录按钮组
    │   ├── SocialButton (Google)           // Google 登录按钮
    │   └── SocialButton (Apple)            // Apple 登录按钮
    └── RegisterForm                        // 注册表单（条件渲染，与 LoginForm 互斥）
        ├── FormTitle                       // "创建您的账户" 标题
        ├── FormField（全名）               // 姓名输入字段
        ├── FormField（邮箱）               // 邮箱输入字段
        ├── FormField（密码）               // 密码输入字段
        ├── FormField（确认密码）            // 确认密码输入字段
        ├── SubmitButton                    // "免费注册" 提交按钮
        └── TermsText                       // "服务条款" 与 "隐私政策" 同意文本
```

### 2.3 数据流概览

```
AuthPage（持有全局状态）
  │
  ├── activeTab: 'login' | 'register'       ← TabSwitcher 上报
  ├── isSubmitting: boolean                  ← SubmitButton 上报
  │
  ├──→ TabSwitcher   props: { activeTab, onTabChange }
  ├──→ LoginForm     props: { isSubmitting, onSubmit }   （visible when activeTab === 'login'）
  ├──→ RegisterForm  props: { isSubmitting, onSubmit }    （visible when activeTab === 'register'）
  └──→ SocialLoginButtons  props: { onGoogleLogin, onAppleLogin }
```

## 三、组件详细拆分

### 3.1 AuthCard（认证卡片容器）

| 属性 | 内容 |
|------|------|
| **组件名** | `AuthCard` |
| **文件路径** | `@/components/auth/auth-card.tsx` |
| **类型** | 服务端组件（纯展示） |
| **核心职责** | 白色圆角卡片容器，包裹所有认证表单内容。提供最大宽度约束和居中布局。 |

**HTML 原型**：
```html
<main class="w-full max-w-[460px]">
  <div class="bg-surface-container rounded-2xl shadow-lg p-8 md:p-10">
    <!-- children -->
  </div>
</main>
```

**样式作用域**：`w-full max-w-[460px]` 控制卡片最大宽度，`bg-surface-container` 对应 `#ffffff`，`rounded-2xl` 对应 `24px` 圆角，`shadow-lg` 对应大阴影，`p-8 md:p-10` 控制内边距。

**Props 接口**：
```typescript
interface AuthCardProps {
  children?: ReactNode;
}
```

### 3.2 TabSwitcher（登录/注册切换标签栏）

| 属性 | 内容 |
|------|------|
| **组件名** | `TabSwitcher` |
| **文件路径** | `@/components/auth/tab-switcher.tsx` |
| **类型** | 客户端组件（`'use client'`） |
| **核心职责** | 渲染登录/注册两个标签按钮，管理激活/非激活视觉状态，点击后回调父组件切换表单。 |

**包含的 HTML 元素**：
- 外层容器：`flex p-1 bg-secondary-container rounded-xl mb-8`（`secondary-container` = `#f1f5f1`，`rounded-xl` = 12px 圆角容器，内间距 4px）
- 登录按钮（激活态）：`flex-1 py-2.5 text-body-md font-medium rounded-lg transition-all duration-300 bg-primary text-on-primary shadow-sm`
- 注册按钮（非激活态）：`flex-1 py-2.5 text-body-md font-medium rounded-lg transition-all duration-300 text-on-surface-variant hover:text-on-surface`

**状态管理**：
- `activeTab` 由父组件 `AuthPage` 通过 Props 传入
- 点击按钮触发 `onTabChange('login')` 或 `onTabChange('register')`
- 按钮激活态切换逻辑在父组件中维护，TabSwitcher 仅根据 props 渲染不同的 className
- 切换时有 `duration-300` 的 CSS 过渡动画

**Props 接口**：
```typescript
interface TabSwitcherProps {
  activeTab: 'login' | 'register';
  onTabChange: (tab: 'login' | 'register') => void;
}
```

### 3.8 SubmitButton（提交按钮）

| 属性 | 内容 |
|------|------|
| **组件名** | `SubmitButton` |
| **文件路径** | `@/components/auth/submit-button.tsx` |
| **类型** | 客户端组件（`'use client'`） |
| **核心职责** | 全宽主色提交按钮。提交时显示"处理中..."文字并禁用交互，模拟 800ms 的 API 延迟反馈。 |

**HTML 原型**（按钮样式）：
```html
<button type="submit"
  class="w-full py-3.5 bg-primary text-on-primary rounded-xl
         font-semibold hover:opacity-90 active:scale-[0.98]
         transition-all shadow-md shadow-primary/10">
  立即登录
</button>
```

**交互逻辑**（来自 HTML 的 `handleAuth` 函数）：
1. 点击提交后，按钮文字变为"处理中..."
2. 按钮变为禁用状态（`disabled = true`）
3. 添加 `opacity-80` 和 `cursor-not-allowed` 类
4. 800ms 后显示 `alert()` 提示，恢复按钮原状

**Props 接口**：
```typescript
interface SubmitButtonProps {
  /** 按钮显示文字（默认"立即登录"或"免费注册"） */
  label: string;
  /** 是否正在提交 */
  isSubmitting: boolean;
  /** 提交中文字（默认"处理中..."） */
  submittingText?: string;
}
```

## 四、自定义 Hooks

### 4.1 useAuthTab

**职责**：管理登录/注册表单的切换状态。提供当前激活标签、切换函数、以及当前表单是否为登录态的判定。

**核心逻辑**（对应 HTML 中 `switchTab` 函数）：
- 状态 `activeTab: 'login' | 'register'`，初始值 `'login'`
- `switchTab(type)` 切换激活态，并触发表单的显隐切换
- 切换时对即将显示的表单添加 `fade-enter-active` 类，300ms 后移除（CSS 动画 `fadeIn`）

```typescript
interface UseAuthTabReturn {
  activeTab: 'login' | 'register';
  switchTab: (tab: 'login' | 'register') => void;
  /** 切换动画是否正在播放 */
  isAnimating: boolean;
}
```

### 4.2 useAuthSubmit

**职责**：处理表单提交流程，管理加载状态，模拟 800ms 的 API 请求延迟。

**核心逻辑**（对应 HTML 中 `handleAuth` 函数）：
1. 阻止表单默认提交行为
2. 设置 `isSubmitting = true`，禁用按钮交互
3. 800ms 后执行回调（可扩展为真实 API 调用）
4. 恢复 `isSubmitting = false`

```typescript
interface UseAuthSubmitReturn {
  isSubmitting: boolean;
  handleSubmit: (event: React.FormEvent, onSuccess?: () => void) => void;
}
```

## 七、样式集成说明

### 7.1 与现有 globals.css 的差异

认证页的设计令牌与官网（landing）和工作区（workspace）存在以下差异，实现**必须在 `globals.css` 中新增认证页专用令牌或覆盖值**：

**需要新增的 CSS 变量**：
```css
:root {
  /* 认证页专用令牌（覆盖 landing 体系的值） */
  --auth-surface-container: #ffffff;      /* 覆盖 --surface-container: #ebefec */
  --auth-secondary-container: #f1f5f1;    /* 覆盖 --secondary-container: #aeedd5 */
  --auth-on-surface-variant: #3e4945;     /* 覆盖 --on-surface-variant: #3e4946 */

  /* 认证页专用字号 */
  --auth-font-size-h1: 32px;
  --auth-font-size-h2: 24px;
  --auth-font-size-body-md: 14px;
  --auth-font-size-label-sm: 12px;
}
```

### 7.2 字体集成

认证页使用 Inter 字体，与官网一致。`app/layout.tsx` 中已添加 Inter 字体的 Google Fonts 链接，无需重复添加。

### 7.3 动画样式

认证页的 tab 切换动画（fadeIn）需要新增全局 CSS：

```css
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(8px); }
  to { opacity: 1; transform: translateY(0); }
}

.fade-enter-active {
  animation: fadeIn 0.3s ease-out;
}
```

### 7.4 输入框焦点环

认证页使用 `focus:ring-2 focus:ring-primary/20 focus:border-primary` 作为输入框聚焦样式。Tailwind CSS 4 的 ring 功能默认可用，无需额外配置。

### 7.5 @tailwindcss/forms 插件

HTML 原型引用了 `@tailwindcss/forms` 插件用于 checkbox 和 input 的基础样式重置。如果项目未安装此插件，可以通过以下方式替代：
- 为 checkbox 手动添加 `appearance-none` + 自定义选中态样式
- 或保留原生 checkbox 样式，仅应用 `w-4 h-4 rounded border-outline-variant text-primary focus:ring-primary cursor-pointer`

## 九、路由规划建议

认证页路由建议使用以下的其中一种方案：

**方案 A（推荐）**：独立路由组 `(auth)`
```
app/
└── (auth)/
    └── login/
        └── page.tsx         → 访问路径 /login
```

**方案 B**：与官网同层级
```
app/
└── login/
    └── page.tsx             → 访问路径 /login
```

推荐方案 A，将认证页面放在独立路由组中，便于后续扩展（如添加 `register`、`reset-password` 等页面）。
