# 认证架构 — 完整分层设计

> **更新日期**：2026-06-03
> **技术栈**：Next.js 16 (App Router) + Supabase Auth + @supabase/ssr v0.10+
> **关联文档**：[04认证组件.md](./04认证组件.md)（UI 组件拆分方案）、[1.sql](./1.sql)（数据库模型）
> **架构原则**：严格七层单向依赖，上层调用下层，下层绝不依赖上层

### 核心依赖规则

| 规则 | 说明 |
|------|------|
| **proxy.ts 先行** | 请求先经过 `proxy.ts`，完成 Session 验证后再到达页面 |
| **状态层统一入口** | 所有组件通过 `useAuth()` 获取认证状态，不直接调 repository |
| **密码永不到达前端** | bcrypt 哈希在 Supabase 服务端完成，前端只传明文密码 |
| **Session 存储在 httpOnly cookie** | 不由前端 JS 直接读写，防 XSS 窃取 |
| **类型安全** | `AuthResponse` 为 discriminated union，禁止 `any` |

## 二、proxy.ts — 入口守卫（Next.js 16 标准模式）

### 2.1 为什么用 proxy.ts 而非 middleware.ts

| 模式 | 状态 | 说明 |
|------|------|------|
| `middleware.ts` | **已废弃** | Next.js 16 中 `@supabase/ssr` 推荐使用 `proxy.ts` |
| `proxy.ts` | **当前标准** | 更好的命名，与 Next.js 内置 proxy 机制对齐 |

### 2.2 执行流程

```
浏览器请求 /workspace
        │
        ▼
  proxy.ts (export async function proxy())
        │
        ├─► createServerClient (读取 request cookies)
        │
        ├─► supabase.auth.getUser()
        │
        ├─► 无 user + 受保护路由？ → 302 redirect → /login
        ├─► 有 user + /login？    → 302 redirect → /workspace
        │
        └─► 正常放行 + cookie 刷新 → NextResponse.next()
```

### 2.3 文件职责

| 文件 | 职责 | 运行环境 |
|------|------|---------|
| `proxy.ts`（根目录） | 导出 `proxy()` 入口函数 + `config.matcher` | Edge Runtime |
| `lib/supabase/proxy.ts` | `updateSession()` — 创建 Server Client、调用 `getUser()`、路由守卫逻辑 | Edge Runtime |

### 2.4 proxy.ts 代码

```typescript
// proxy.ts — 项目根目录
import { updateSession } from '@/lib/supabase/proxy'
import type { NextRequest } from 'next/server'

export async function proxy(request: NextRequest) {
  return await updateSession(request)
}

export const config = {
  matcher: [
    // 排除静态资源，其余所有请求都走 proxy
    '/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)',
  ],
}
```

```typescript
// lib/supabase/proxy.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

export async function updateSession(request: NextRequest) {
  let supabaseResponse = NextResponse.next({ request })

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY!,
    {
      cookies: {
        getAll() { return request.cookies.getAll() },
        setAll(cookiesToSet) {
          for (const { name, value } of cookiesToSet) {
            request.cookies.set(name, value)
          }
          supabaseResponse = NextResponse.next({ request })
          for (const { name, value, options } of cookiesToSet) {
            supabaseResponse.cookies.set(name, value, options)
          }
        },
      },
    }
  )

  const { data: { user } } = await supabase.auth.getUser()

  const protectedPaths = ['/workspace', '/trash']
  const isProtectedRoute = protectedPaths.some((p) => request.nextUrl.pathname.startsWith(p))
  const isAuthRoute = request.nextUrl.pathname === '/login'

  if (!user && isProtectedRoute) {
    return NextResponse.redirect(new URL('/login', request.url))
  }
  if (user && isAuthRoute) {
    return NextResponse.redirect(new URL('/workspace', request.url))
  }

  return supabaseResponse
}
```

### 2.5 路由守卫矩阵

| 用户状态 | 访问 `/login` | 访问 `/workspace`, `/trash` | 访问 `/`（官网） |
|----------|-------------|---------------------------|---------------|
| 未登录 | 正常显示 | **302 → /login** | 正常显示 |
| 已登录 | **302 → /workspace** | 正常显示 | 正常显示 |

## 四、类型定义层

### 4.1 types/auth.ts

```typescript
// 与 Supabase Auth User 对齐，附加业务字段
export interface User {
  id: string          // UUID（Supabase Auth 统一使用 uuid）
  uid: string         // 公开短 ID
  email: string
  phone?: string
  fullName: string    // 映射 DB: full_name
  avatar: string      // 首字字符
  membership: 'free' | 'pro' | 'enterprise'
  createdAt: string
  updatedAt: string
}

export interface LoginPayload {
  email: string
  password: string
  rememberMe?: boolean
}

export interface RegisterPayload {
  fullName: string
  email: string
  password: string
}

// ✦ Discriminated union — success 时必有 user，error 时必有 error
export type AuthResponse =
  | { success: true; user: User }
  | { success: false; error: string }
```

### 4.2 前端类型 ↔ Supabase 字段映射

| types/auth.ts | Supabase Auth User | 说明 |
|--------------|-------------------|------|
| `id` | `user.id` | UUID 直接映射 |
| `uid` | `user_metadata.uid` | 自定义，注册时写入 |
| `email` | `user.email` | Supabase 管理 |
| `fullName` | `user_metadata.full_name` | 自定义，snake_case → camelCase |
| `avatar` | `user_metadata.avatar` | 自定义，首字字符 |
| `phone` | `user_metadata.phone` | 自定义 |
| `membership` | `user_metadata.membership` | 自定义，默认 'free' |
| `createdAt` | `user.created_at` | Supabase 自动 |
| `updatedAt` | `user.updated_at` | Supabase 自动 |

**永远不返回给前端**：`password`, `password_hash`, `encrypted_password`

### 5.2 mapUser — Supabase User → 前端 User

```typescript
// repositories/supabase-auth.ts
private mapUser(supabaseUser: SupabaseUser): User {
  const metadata = supabaseUser.user_metadata ?? {}
  return {
    id: supabaseUser.id,
    uid: metadata.uid ?? supabaseUser.id.slice(0, 8),
    email: supabaseUser.email ?? '',
    phone: metadata.phone ?? undefined,
    fullName: metadata.full_name ?? '',
    avatar: (metadata.full_name as string)?.charAt(0) ?? '用',
    membership: metadata.membership ?? 'free',
    createdAt: supabaseUser.created_at,
    updatedAt: supabaseUser.updated_at ?? supabaseUser.created_at,
  }
}
```

## 六、状态层（React Context）

### 6.1 AuthProvider 数据流

```
AuthProvider (root layout)
  │
  ├─ mount 时:
  │    supabaseAuthRepo.getSession() → setUser(sessionUser)
  │    supabaseAuthRepo.onAuthStateChange() → 监听多标签页同步
  │
  ├─ login(email, password):
  │    1. supabaseAuthRepo.signIn(email, password)
  │    2. 成功 → setUser(result.user)
  │    3. 失败 → setAuthError(result.error)
  │    4. 返回 { success, error? }
  │
  ├─ register(payload):
  │    1. supabaseAuthRepo.signUp(payload)           ← 注册时自动生成 uid 并写入 user_metadata
  │    2. DB trigger handle_new_user() 从 user_metadata.uid 读取并写入 profiles
  │    3. 成功 → setUser(result.user)
  │    4. 失败 → setAuthError(result.error)
  │
  ├─ signInWithOAuth(provider):
  │    1. supabaseAuthRepo.signInWithOAuth(provider)  ← 页面跳转至 Google/Apple 授权
  │    2. 授权成功后 Supabase 自动回调 redirectTo URL
  │
  ├─ logout():
  │    1. supabaseAuthRepo.signOut()
  │    2. setUser(null)                              ← 清除全局状态
  │
  ├─ sendPasswordReset(email):
  │    1. supabaseAuthRepo.sendPasswordReset(email)  ← 调用 Supabase resetPasswordForEmail
  │    2. 返回 { success, error? }
  │
  ├─ updateUser(updates):
  │    1. setUser(prev → { ...prev, ...updates })    // 先更新本地 UI
  │    2. supabaseAuthRepo.updateProfile(updates)    // 同步 auth.users + public.profiles
  │
  ├─ changePassword(newPassword):
  │    1. supabaseAuthRepo.updatePassword(newPassword)
  │    2. 返回 boolean
  │
  └─ deleteAccount():
       1. supabaseAuthRepo.deleteAccount(user.id)    ← 调用 delete_user_account RPC 软删除
       2. supabaseAuthRepo.signOut()                 ← 清除 session
       3. setUser(null)                              ← 清除全局状态
```

## 九、认证流程时序图

### 9.1 登录流程

```
用户填写邮箱/密码 → 点击"登录"
        │
        ▼

        │
        ▼
auth-page.tsx → handleLoginSubmit()
        │
        ▼
useAuth().login(email, password)
        │
        ▼
supabaseAuthRepo.signIn(email, password)
        │
        ▼
supabase.auth.signInWithPassword()   ← Supabase JS SDK
        │
        ▼
Supabase 服务端:
  - 验证邮箱 + bcrypt 比对 password_hash
  - 签发 JWT access_token + refresh_token
  - 设置 httpOnly cookie（通过 proxy.ts 写入）
        │
        ▼
返回 { data: { user, session } }
        │
        ▼
mapUser(data.user) → 前端 User 类型
        │
        ▼

        │
        ▼

```

### 9.2 注册流程

```
用户填写信息 → 点击"注册"
        │
        ▼
supabaseAuthRepo.signUp(payload)
        │  - generateUid() → 8 位 hex 字符串
        │  - 写入 options.data.uid
        │
        ▼
supabase.auth.signUp({
  email, password,
  options: { data: { full_name, uid } }
})
        │
        ▼
Supabase 服务端:
  - 检查 email 唯一
  - bcrypt 生成 password_hash
  - 写入 auth.users 表
  - 触发 handle_new_user() trigger:
      ├─ 读取 user_metadata.uid → 写入 profiles.uid
      ├─ 读取 user_metadata.full_name → 写入 profiles.full_name
      └─ 创建根文件夹"我的文档"
        │
        ▼
返回 user → 自动登录 → router.push('/workspace')
```

### 9.3 登出流程

```
用户点击"退出登录"
        │
        ▼
useAuth().logout()
        │
        ▼
supabaseAuthRepo.signOut()
        │
        ▼
supabase.auth.signOut()
  - Supabase 清除服务端 session
  - Cookie 被标记过期
        │
        ▼
setUser(null) → React re-render
router.push('/login')
```

### 9.4 修改 Profile 流程（含侧边栏同步 + DB 双写）

```
用户修改姓名/手机/头像
        │
        ▼
AccountTab → updateUser({ fullName: '新名' })
        │
        ├─► 1. setUser(prev → { ...prev, ...updates })  ← 即时 UI 更新
        │       → AuthProvider 重新渲染
        │       → UserSection（workspace 侧边栏）通过 useAuth().user 自动同步
        │

                  │
                  ├─► supabase.auth.updateUser({ data })
                  │      └─ 写入 auth.users.raw_user_metadata 字段
                  │
                  └─► supabase.from('profiles').update(...)
                         └─ 同步写入 public.profiles 表
                              下次 getSession() / onAuthStateChange 自动带回
```

## 十、安全要点

| 安全项 | 措施 |
|--------|------|
| **密码存储** | Supabase 服务端 bcrypt 哈希，前端永不接触 hash |
| **Session 存储** | httpOnly cookie（通过 `@supabase/ssr` 管理），不可被 JS 读取 |
| **CSRF 防护** | Supabase Auth 内置 CSRF token 机制 |
| **XSS 防护** | User 对象不包含敏感字段（无 password_hash），localStorage 仅存搜索记录 |
| **速率限制** | Supabase Auth 服务端内置登录频率限制 |
| **Token 过期** | JWT 短期 access_token (1h) + 长期 refresh_token，proxy.ts 自动刷新 |
| **多设备** | `onAuthStateChange` 监听跨标签页同步，登出一处即全部登出 |

### 11.3 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 登录后无限重定向 | proxy.ts 中 cookie 未正确设置 | 检查 `setAll` 的 `request.cookies.set` + `supabaseResponse` 双重写入 |

| OAuth 回调 404 | 未在 Supabase 配置 redirect URL | 在 Supabase Dashboard → Authentication → URL Configuration 添加回调地址 |
| `deleteAccount` RPC 报错 | `delete_user_account` 函数未创建 | 在 Supabase SQL Editor 执行 `docs/1.sql` 中的函数定义 |
