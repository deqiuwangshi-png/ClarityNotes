# 搜索栏模块

> 侧边栏文档搜索：标题匹配 + 正文摘要 + 实时过滤 + 独立结果视图。
> 本文档包含架构设计、分层方案、数据流与 AI 实现约定。
> 更新时间：2026-06-02

---

## 一、需求概述

| 维度 | 描述 |
|------|------|
| 搜索范围 | 文件树中的所有文档（`type === "file"`），标题匹配 + 模糊匹配 |
| 结果项 | 文档标题（高亮匹配词）、面包屑路径、正文摘要（匹配词附近上下文） |
| 触发方式 | 实时过滤（输入后防抖 300ms） |
| 结果位置 | 左侧侧边栏内替换原有文件树区域 |
| 点击结果 | 右侧内容区加载对应文档编辑视图，左侧不关闭搜索结果 |
| 关闭搜索 | 点击返回按钮 / 清空搜索框 → 恢复文件树，右侧保持当前文档 |
| 空状态 | 显示"未找到相关文档" |

## 二、架构分层

```
┌─────────────────────────────────────────────────────┐
│  UI 层                                              │
│  SearchBar.tsx ← 输入事件派发                        │
│  SearchResultsView.tsx ← 结果列表 + 空状态 + 返回按钮 │
│  SidebarLayout.tsx ← 搜索态切换（FileTree ⇄ 搜索结果） │
└────────────────────┬────────────────────────────────┘
                     │ useSearchStore()
┌────────────────────▼────────────────────────────────┐
│  状态层                                             │
│  store/searchStore.ts                               │
│  state:  query / results / isSearching / debounce    │
│  actions: setQuery / clearSearch                     │
└────────────────────┬────────────────────────────────┘
                     │ 调用 searchNodes()
┌────────────────────▼────────────────────────────────┐
│  业务逻辑层                                         │
│  lib/services/searchService.ts                      │
│  searchNodes(tree, query): SearchResultItem[]        │
│  extractSnippet(content, query): string              │
│  getNodePath(tree, nodeId): BreadcrumbItem[]         │
└────────────────────┬────────────────────────────────┘
                     │ 读取 tree（来自 fileTreeStore）
┌────────────────────▼────────────────────────────────┐
│  数据层                                             │
│  无新增 — 直接读取 fileTreeRepo.getTree()            │
└─────────────────────────────────────────────────────┘
```

### 依赖规则

- 搜索不创建新的 Repository，不写数据，只读文件树
- `searchStore` 通过 `useFileTreeStore.getState().tree` 获取树
- `searchService` 是纯函数：输入 `TreeNode[] + query`，输出 `SearchResultItem[]`
- `SidebarLayout` 渲染 `SearchResultsView` 时传入 `results` 和回调

## 三、新增类型

```ts
// types/fileTree.ts（追加）

interface SearchResultItem {
  nodeId: string              // 匹配到的文档 TreeNode.id
  title: string               // 文档标题（如 "需求分析.md"）
  path: BreadcrumbItem[]      // 面包屑路径（从根到该文档父目录）
  snippet: string             // 匹配词所在上下文的摘要片段（最多 50 字符）
}
```

## 五、数据流（详细）

### 5.1 搜索触发

```
[UI] SearchBar onChange
  → searchStore.setQuery(value)  ← 防抖 300ms 内置
    → 300ms 后执行搜索:
      → useFileTreeStore.getState().tree
      → searchService.searchNodes(tree, query)  ← 纯函数
        → 遍历树找所有 file 节点
        → 标题匹配（不区分大小写）
        → 匹配到的: extractSnippet(node.content, query)
        → getNodePath(tree, node.id)
        → 返回 SearchResultItem[]
      → searchStore: set({ results, isSearching: true })
```

### 5.2 视图切换

```
SidebarLayout:
  searchStore.isSearching && query !== ""
    ? <SearchResultsView
        results={results}
        onClear={() => searchStore.clearSearch()}
        onSelect={(nodeId) => {
          fileTreeStore.selectNode(nodeId);
          // 不关闭搜索，右侧打开文档
        }}
      />
    : <>
        <SearchBar />
        <FileTree />
      </>
```

### 5.3 关闭搜索

```
[UI] 返回按钮 onClick / 清空搜索框
  → searchStore.clearSearch()
    → set({ query: "", results: [], isSearching: false })

右侧编辑器保持不变（不走 restoreFromTrash 之类的重置）
```

## 六、searchService 函数签名

```ts
// lib/services/searchService.ts

searchNodes(tree: TreeNode[], query: string): SearchResultItem[]
  // 遍历树，对所有 file 节点做标题匹配
  // 标题包含 query 即命中（不区分大小写）
  // 对每个命中项生成 path 和 snippet

extractSnippet(htmlContent: string | undefined, query: string): string
  // 从 HTML 正文中去除标签
  // 找到 query 首次出现位置
  // 取前后共 50 字符的上下文片段

getNodePath(tree: TreeNode[], targetNodeId: string): BreadcrumbItem[]
  // 获取该文档在树中的完整面包屑路径
  // 不含目标节点自身，只到父级
```

## 七、SearchResultsView UI 约定

```
┌────────────────────────────────────┐
│ ← 返回       共 3 条搜索结果       │  ← 顶部栏
├────────────────────────────────────┤
│ 需求分析.md               ↓ 新页面│  ← 结果项 1
│ 我的文档 > 项目产品               │   面包屑
│ ...个人知识操作系统...             │   摘要
├────────────────────────────────────┤
│ 竞品调研.md               ↓ 新页面│  ← 结果项 2
│ 我的文档 > 产品调研               │
│ ...市面上有 3 种主流...           │
├────────────────────────────────────┤
│                                    │  ← 空状态（无结果时）
│        未找到相关文档              │
│                                    │
└────────────────────────────────────┘
```

**UI 实现关键点（遵守 [09UI定制.md](./09UI定制.md)）：**

1. 匹配词高亮：使用 `<mark>` 标签，背景 `bg-mint-hover`（#E4F5EF），圆角 `rounded`
2. 面包屑路径使用现有 `<Breadcrumb>` 组件或内联文本
3. 空状态居中显示图标 + 文字
4. 所有颜色使用薄荷绿令牌体系
5. 点击结果项 → `onSelect(nodeId)` → `fileTreeStore.selectNode(nodeId)`
6. "返回" 按钮用 Material Icons `arrow_back`

## 八、AI 实现约定

### 8.1 只能修改/创建以下文件

| 文件 | 操作 |
|------|------|
| `types/fileTree.ts` | 追加 `SearchResultItem` |
| `lib/services/searchService.ts` | **新建** |
| `store/searchStore.ts` | **新建** |
| `components/workspace/sidebar/SearchResultsView.tsx` | **新建** |
| `components/workspace/sidebar/SearchBar.tsx` | 修改（props → store） |
| `components/workspace/layout/SidebarLayout.tsx` | 修改（条件渲染） |

### 8.2 禁止事项

- ❌ 不新建 Repository
- ❌ 不修改 `globals.css`
- ❌ 不修改任何 `@theme` 令牌
- ❌ 不修改 `EditorBody`、`DocumentEditor`、`FolderView`
- ❌ 不修改 `fileTreeStore` 的现有 action
- ❌ 不引入新依赖包

### 8.3 实现顺序

1. `types/fileTree.ts` — 类型先行
2. `lib/services/searchService.ts` — 纯函数
3. `store/searchStore.ts` — 状态
4. `components/workspace/sidebar/SearchResultsView.tsx` — UI 新组件
5. `components/workspace/sidebar/SearchBar.tsx` — 去 props 化
6. `components/workspace/layout/SidebarLayout.tsx` — 条件渲染

### 8.4 技术细节

- 防抖：在 `searchStore.setQuery()` 内部用 `setTimeout` 实现，300ms
- `extractSnippet`：用正则 `/<[^>]*>/g` 去除 HTML 标签，再用 `indexOf` + `substring` 截取
- `getNodePath`：复用 `buildBreadcrumb`（fileTreeService 已有），取最后一个之前的部分（去掉文档自身）
- 搜索范围：**只搜索 `type === "file"` 的节点**，不搜索文件夹
