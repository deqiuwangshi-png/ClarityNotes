# UI 定制保护约定

> **核心原则：项目的所有 UI 样式已经是最终设计，严禁任何修改。**
>
> 本文档是 AI 助手的硬性约束，违反任一条款均视为严重错误。
> 更新时间：2026-06-02

---

## 一、总则

### 1.1 适用范围

以下**所有**方面均属于 "UI 定制"，AI **禁止**修改：

| 类别 | 具体内容 | 涉及范围 |
|------|----------|----------|
| **颜色** | 主色、背景色、文字色、边框色、悬停色、状态色 | 所有组件与页面 |
| **圆角** | `rounded-*`、`border-radius` CSS 属性 | 卡片、按钮、输入框、弹窗、菜单 |
| **阴影** | `shadow-*`、`box-shadow` CSS 属性 | 卡片、侧边栏、弹窗、浮动元素 |
| **字体** | 字体族、字号、字重、行高、字间距 | 标题、正文、标签、按钮文字 |
| **间距** | `padding`、`margin`、`gap`、布局间距 | 卡片内边距、段落间距、组件间距 |
| **动画** | `transition`、`animation`、`transform` | 页面切换、侧边栏折叠、悬停效果 |
| **布局** | `flex`/`grid` 结构、`width`/`height`、位置 | 页面框架、侧边栏宽度、卡片宽度 |
| **边框** | `border` 宽度、颜色、样式 | 分割线、卡片边框、输入框边框 |
| **响应式** | `md:`/`lg:` 断点、响应式布局策略 | 所有组件 |
| **滚动条** | 宽度、颜色、圆角 | 全局样式 |
| **文案** | 按钮文字、标题文字、占位符文字 | 所有 UI 文字（非文档内容） |

### 1.2 铁律

```
项目 UI 样式 === 最终版本
任何形式的"美化"、"优化"、"调整"、"统一" === 禁止
唯一例外：用户主动要求修改某个 UI 元素
```

## 三、受保护组件与样式清单

### 3.1 全局样式 — 绝对不可修改

| 文件 | 受保护内容 |
|------|-----------|
| [app/globals.css](../app/globals.css) | 全部：颜色令牌、字体栈、滚动条样式、`prose-custom`、`editor-card`、`.material-symbols-outlined`、`clarity-editor`、所有 `@theme` 块 |

### 3.2 UI 原子组件 — 任何改动需用户确认

| 组件 | 受保护的样式属性 |
|------|-----------------|
| `Modal` | 尺寸映射(`sm`/`md`/`lg`/`xl`)、圆角(`rounded-2xl`)、背板模糊度(`blur-sm`)、遮罩透明度(`bg-black/40`) |
| `Button` | 三种变体的颜色/边框/悬停效果 |
| `FormInput` | 边框颜色(`mint-border`)、聚焦态、错误态样式 |
| `Checkbox` | 勾选状态颜色(`mint-accent`)、尺寸 |
| `SubmitButton` | 加载动画样式 |
| `Toast` | 位置、动画、颜色 |
| `FloatingMenu` | 圆角、阴影、Portal 渲染方式 |

### 3.3 布局组件 — 结构不可修改

| 组件 | 受保护的样式属性 |
|------|-----------------|
| `SidebarLayout` | 侧边栏宽度(`280px`)、折叠动画参数(`translateX`、`cubic-bezier`)、折叠顶部栏高度(`h-14`)、路由区分逻辑 |
| `AuthGuard` | 无 UI，仅逻辑 |
| `AuthLayout` | 无 UI，仅逻辑 |

### 3.4 编辑器组件 — 排版不可修改

| 组件 | 受保护的样式属性 |
|------|-----------------|
| `EditorBody` | 标题字号(`text-4xl`)、`prose-custom` 排版(line-height 1.8, paragraph margin 1.25rem)、`editor-card` 卡片样式、`max-w-[800px]` 正文宽度 |
| `EditorHeader` | 高度(`h-14`)、背景(`bg-mint-bg`)、边框(`border-mint-border/20`) |
| `BubbleMenu` | 深色背景(`bg-[#1e2522]`)、圆角、尺寸、箭头位置 |

### 3.5 认证页组件

| 组件 | 受保护的样式属性 |
|------|-----------------|

| `AuthForm` | 表单间距、错误提示样式 |
| `AuthCard` | 卡片圆角、阴影、宽度 |

| `SocialLogin` | 分割线样式、按钮样式 |

### 3.6 用户菜单与设置

| 组件 | 受保护的样式属性 |

## 四、允许的修改 vs 禁止的修改

### 4.1 允许（无需用户额外确认）

| 场景 | 示例 |
|------|------|
| 新增 props，不改默认样式 | 给组件加 `className?: string` 合并外部类名 |
| 添加逻辑，不改样式 | 加 `onClick`、状态管理、键盘事件 |
| 新增组件，复用现有 token | 创建新组件，使用 `bg-mint-bg`、`text-mint-text` 等现有令牌 |
| 修复逻辑 BUG | 修正事件处理、状态更新、数据流 |
| 用 `createPortal` 修复定位问题 | 将弹窗渲染到 `document.body` |
| 增加 `data-*` 属性 | 为选区 Hook 添加自定义属性 |
| 条件性渲染 | 根据状态 `true`/`false` 渲染不同内容 |

### 4.2 禁止（必须拒绝或请求用户确认）

| 场景 | 违规示例 |
|------|----------|
| 改颜色 | `text-mint-text` → `text-gray-900` |
| 改圆角 | `rounded-2xl` → `rounded-lg` |
| 改阴影 | `shadow-2xl` → `shadow-xl` |
| 改字号 | `text-4xl` → `text-3xl` |
| 改 padding/margin | `px-6 py-4` → `px-4 py-2` |
| 改宽度/高度 | `max-w-[800px]` → `max-w-[720px]` |
| 改动画参数 | `duration-250` → `duration-300` |
| 删 CSS 类 | 移除 `.prose-custom` 的 `line-height: 1.8` |
| 改设计令牌色值 | 修改 `--color-mint-accent` 的颜色值 |
| 改字体栈 | 修改 `font-family` |
| 改布局结构 | 将 `flex flex-col` 改成 `grid` |
| 改边框样式 | `border-mint-border/30` → `border-gray-200` |

## 五、违规识别清单

当 AI 输出包含以下操作时，立即判定为 UI 污染：

1. ✅ 修改了 `globals.css` 中 `@theme` 块内的任何值
2. ✅ 修改了 `globals.css` 中 `.prose-custom`、`.editor-card`、`.clarity-editor` 的定义
3. ✅ 修改了任何组件的 `className` 中涉及颜色、圆角、阴影、字号、间距的部分
4. ✅ 在组件中引入了新的 Tailwind 颜色类（如 `text-blue-500`、`bg-gray-100`）替代现有 mint 令牌
5. ✅ 修改了 `SidebarLayout` 的宽度值或动画参数
6. ✅ 修改了 Modal 的 `widthMap` 尺寸映射
7. ✅ 修改了 `EditorBody` 的 `max-w-[800px]` 宽度限制
8. ✅ 使用了 Tailwind 任意值语法（如 `text-[15px]`）引入新字号，除非项目已有前例

## 六、例外流程

如果确实需要修改 UI 样式：

1. **用户主动要求** → 严格按用户描述修改，不多改一行
2. **新增页面/组件** → 使用已有令牌，不发明新颜色/间距
3. **无法判断** → 使用 `AskUserQuestion` 工具询问用户是否允许
