# 回收站模块 - React 组件拆分方案

> **技术栈**: Next.js 16 (App Router) + React 19 + TypeScript + Tailwind CSS 4
> **路径别名**: `@/*` 映射至项目根目录
> **设计系统**: 基于 `mint-*` 配色体系（与工作区 workspace 共享同一套令牌），Inter 字体排版，Material Symbols 图标库，自定义复选框样式。

---

## 一、设计令牌清单

回收站模块**复用工作区的 `mint-*` 配色体系**，不引入新颜色令牌。以下为 HTML 原型中实际使用到的令牌值：

| CSS 变量 | 色值 | 用途 |
|----------|------|------|
| `--mint-bg` | `#f6faf7` | 页面主背景 |
| `--mint-text` | `#181d1b` | 主要文字色 |
| `--mint-muted` | `#3e4945` | 次要文字色（如删除时间、文件大小） |
| `--mint-border` | `#bdc9c4` | 边框色（控制透明度层级：`/10`, `/20`, `/30`, `/40`） |
| `--mint-accent` | `#006856` | 主色（选中态、强调元素） |
| `--mint-accent-light` | `#007A66` | 浅主色（底部导航选中态） |
| `--mint-hover` | `#E4F5EF` | 悬停/选中行背景色 |
| `--color-gray-400` | `#9CA3AF` | 列标题文字色、文档图标色 |
| `--color-red-600` | `#dc2626` | 删除按钮文字色（清空/彻底删除） |

**令牌支持**：以上变量已通过 `globals.css` 的 `@theme inline` 暴露为 `--color-mint-bg`、`--color-mint-text` 等 Tailwind 类名，组件中可直接使用 `bg-mint-bg`、`text-mint-text` 等。

## 二、整体页面架构

### 2.1 页面位置

回收站页面是工作区（workspace）的一个子页面，通过 App Router 路由 `/trash` 访问。页面复用了工作区的侧边栏布局，主内容区域替换为回收站内容。

### 2.2 页面级组件

| 组件名 | 文件路径 | 类型 | 核心职责 |
|--------|----------|------|---------|
| `TrashPage` | `@/app/trash/page.tsx` | 客户端页面 | 回收站入口，组合侧边栏与回收站主内容。管理多选模式状态（`isBatchMode`）、选中项列表、全选/取消逻辑。 |

### 2.3 组件层级树

```
TrashPage                                   // 页面入口 - 管理多选状态
├── Sidebar (复用)                          // 侧边栏（与工作区共用，不在此文档描述）
│   ├── UserProfile (复用)
│   ├── SearchBar (复用)
│   ├── FileTree (复用)
│   └── SidebarFooter (复用，回收站高亮)
│
├── TrashHeader                             // 回收站标题栏
│   ├── HeaderTitle                        // "回收站" 标题
│   ├── HeaderDescription                  // 说明文字
│   └── EmptyAllButton                     // "清空回收站" 按钮
│
├── TrashTableHeader                       // 列表列标题行
│   └── MasterCheckbox                     // 全选复选框（多选模式下可见）
│
├── TrashItem[]                            // 回收站列表项
│   ├── BatchCheckbox                      // 单项复选框（多选模式下可见）
│   ├── ItemIcon                           // 文件/文件夹图标
│   ├── ItemInfo                           // 文件名（删除线）+ 原位置
│   ├── ItemMeta                           // 删除时间 + 文件大小
│   └── TrashItemDropdown                  // 更多操作下拉菜单
│       ├── BatchModeTrigger               // "多选" 触发器
│       ├── RestoreTrigger                 // "恢复文件" 
│       └── PermanentDeleteTrigger         // "彻底删除"
│
└── FloatingBatchBar                       // 底部浮动批量操作栏
    ├── CountBadge                         // 选中数量徽标
    ├── CancelButton                       // "取消" 退出多选
    ├── BatchRestoreButton                 // "恢复" 批量恢复
    └── BatchDeleteButton                  // "确认删除" 批量删除
```

### 2.4 数据流概览

```
TrashPage（持有全局状态）
  │
  ├── isBatchMode: boolean                 ← 由 TrashItemDropdown 的 "多选" 触发
  │                                           或 FloatingBatchBar 的 "取消" 关闭
  ├── selectedIds: Set<string>             ← 勾选的文档 ID 集合
  ├── isAllSelected: boolean               ← 是否全选
  ├── isIndeterminate: boolean             ← 是否半选状态
  │
  ├──→ TrashHeader         props: { onEmptyAll }
  ├──→ TrashTableHeader    props: { isBatchMode, isAllSelected, isIndeterminate, onToggleSelectAll }
  ├──→ TrashItem[]         props: { item, isBatchMode, isSelected, onSelect, onEnterBatch, onRestore, onPermanentDelete }
  └──→ FloatingBatchBar    props: { selectedCount, isVisible, onCancel, onBatchRestore, onBatchDelete }
```

**Props 接口**：
```typescript
interface TrashHeaderProps {
  onEmptyAll?: () => void;
}
```

### 3.2 TrashTableHeader（列标题行）

| 属性 | 内容 |
|------|------|
| **组件名** | `TrashTableHeader` |
| **文件路径** | `@/components/trash/trash-table-header.tsx` |
| **类型** | 客户端组件 |
| **核心职责** | 渲染表格列标题：文件名/原位置、删除时间、文件大小、操作。多选模式下展开全选复选框，支持全选/取消和半选（indeterminate）状态。 |

**HTML 原型**：
```html
<div id="table-grid-header" class="mb-3 grid grid-cols-[1fr_200px_140px_80px] items-center gap-4 px-5 text-xs font-medium text-[#9CA3AF]">
  <span class="flex items-center min-w-0">
    <div id="header-checkbox-wrapper" class="w-0 opacity-0 mr-0 transition-all duration-200 overflow-hidden flex items-center h-4">
      <input type="checkbox" id="select-all-checkbox" onchange="toggleSelectAll(this)" class="custom-checkbox" />
    </div>
    <span>文件名 / 原位置</span>
  </span>
  <span class="text-right">删除时间</span>
  <span class="text-right">文件大小</span>
  <span></span>
</div>
```

**样式作用域**：`grid grid-cols-[1fr_200px_140px_80px]` 四列网格。全选复选框使用 `custom-checkbox` 类（薄荷绿自定义复选框样式，详见 CSS）。复选框容器有展开/收缩过渡动画（`w-0 opacity-0 mr-0` ↔ `w-4 opacity-100 mr-3`，`transition-all duration-200`）。

**交互逻辑**（对应 HTML 中 `toggleSelectAll` 函数）：
- 点击全选复选框，遍历所有 `TrashItem` 设置选中/取消
- 根据子项选中数量自动切换三种状态：
  - 全部未选 → 未选中（`checked=false, indeterminate=false`）
  - 部分选中 → 半选（`checked=false, indeterminate=true`）
  - 全部选中 → 全选（`checked=true, indeterminate=false`）

**Props 接口**：
```typescript
interface TrashTableHeaderProps {
  isBatchMode: boolean;
  isAllSelected: boolean;
  isIndeterminate: boolean;
  onToggleSelectAll: () => void;
}
```

### 3.3 TrashItem（回收站列表项）

| 属性 | 内容 |
|------|------|
| **组件名** | `TrashItem` |
| **文件路径** | `@/components/trash/trash-item.tsx` |
| **类型** | 客户端组件 |
| **核心职责** | 渲染单行回收站文件/文件夹。包含图标、名称（删除线）、原位置、删除时间、文件大小和更多操作下拉菜单。支持多选模式下展开复选框和行高亮。 |

**HTML 原型**：
```html
<div class="row-item group grid grid-cols-[1fr_200px_140px_80px] w-full items-center gap-4 rounded-xl bg-white px-5 py-4 shadow-sm border border-mint-border/10 transition-all duration-200 hover:border-mint-border/30 hover:shadow-md cursor-pointer"
     onclick="handleRowClick(this)">
  <!-- 复选框区域（多选模式下可见） -->
  <span class="flex items-center min-w-0">
    <div class="checkbox-wrapper w-0 opacity-0 mr-0 transition-all duration-200 overflow-hidden flex items-center h-4"
         onclick="event.stopPropagation();">
      <input type="checkbox" class="custom-checkbox item-checkbox" onchange="handleItemCheckboxChange()" />
    </div>
    <!-- 图标 + 名称 + 原位置 -->
    <span class="flex items-center gap-3.5 min-w-0">
      <span class="flex size-9 shrink-0 items-center justify-center rounded-xl bg-gray-100 text-[#9CA3AF]">
        <span class="material-symbols-outlined filled text-[20px]">folder</span>
        <!-- 文档使用: <span class="material-symbols-outlined text-[20px]">description</span> -->
      </span>
      <div class="flex flex-col min-w-0">
        <span class="truncate text-sm font-medium text-mint-text/80 line-through decoration-gray-300">旧版设计参考</span>
        <span class="text-[11px] text-gray-400 truncate mt-0.5">原位置: 我的文档/设计资源</span>
      </div>
    </span>
  </span>
  <span class="text-right text-xs text-mint-muted">2026-05-28 10:12</span>
  <span class="text-right text-xs text-mint-muted">—</span>
  <!-- 操作下拉菜单 -->
  <div class="relative flex justify-end dropdown-container" onclick="event.stopPropagation();">
    <button onclick="toggleDropdown(event, 'item-dropdown-1')"
            class="flex size-8 items-center justify-center rounded-full text-mint-muted transition hover:bg-mint-hover hover:text-mint-accent opacity-0 group-hover:opacity-100 focus:opacity-100">
      <span class="material-symbols-outlined text-[18px]">more_horiz</span>
    </button>
    <!-- 下拉菜单：TrashItemDropdown -->
  </div>
</div>
```

**样式状态**：
- 默认：`bg-white`，`border-mint-border/10`，`shadow-sm`
- 悬停：`hover:border-mint-border/30`，`hover:shadow-md`
- 选中（多选高亮）：`bg-mint-hover/40`，`border-mint-border/40`
- 文件名有删除线：`line-through decoration-gray-300`
- 原位置使用 `text-[11px] text-gray-400`

**交互逻辑**（对应 HTML 中 `handleRowClick` 函数）：
- 非多选模式：点击行触发打开/预览文件
- 多选模式：点击行切换复选框选中状态，联动高亮行

**Props 接口**：
```typescript
interface TrashItemData {
  id: string;
  type: 'folder' | 'file';
  name: string;
  originalPath: string;
  deletedAt: string;
  size: string;
}

interface TrashItemProps {
  item: TrashItemData;
  isBatchMode: boolean;
  isSelected: boolean;
  onSelect: (id: string) => void;
  onEnterBatch: (itemId: string) => void;
  onRestore: (itemId: string) => void;
  onPermanentDelete: (itemId: string) => void;
}
```

- 下拉菜单由 `useTrashDropdown` Hook 控制显隐
- "多选"选项触发批量模式入口，自动勾选当前行
- 点击外部区域自动关闭下拉菜单（对应 HTML 的 `document.addEventListener('click', ...)`）
- 带边框分隔线：`my-1 border-t border-gray-100`

**Props 接口**：
```typescript
interface TrashItemDropdownProps {
  itemId: string;
  isOpen: boolean;
  onToggle: () => void;
  onClose: () => void;
  onEnterBatch: (itemId: string) => void;
  onRestore: (itemId: string) => void;
  onPermanentDelete: (itemId: string) => void;
}
```

- 边框：`border-[#2b3531]`（深色边框）

## 四、自定义 Hooks

### 4.1 useBatchMode

**职责**：管理回收站的多选模式状态。包括进入/退出多选模式、全选/取消、单项选中/取消、选中计数、行高亮等逻辑。

**核心逻辑**（对应 HTML 中 `isBatchMode` / `enterBatchMode` / `exitBatchMode` / `handleItemCheckboxChange` / `toggleSelectAll` / `toggleRowHighlight`）：

```typescript
interface UseBatchModeReturn {
  isBatchMode: boolean;
  selectedIds: Set<string>;
  isAllSelected: boolean;
  isIndeterminate: boolean;
  selectedCount: number;
  enterBatchMode: (initialItemId?: string) => void;
  exitBatchMode: () => void;
  toggleSelectAll: () => void;
  toggleSelectItem: (itemId: string) => void;
}
```

**状态关系**：
| 选中状态 | selectedIds | isAllSelected | isIndeterminate |
|----------|-------------|---------------|-----------------|
| 全部未选 | 空 `Set` | `false` | `false` |
| 部分选中 | 部分 ID | `false` | `true` |
| 全部选中 | 全部 ID | `true` | `false` |

### 4.2 useTrashDropdown

**职责**：管理单个列表项下拉菜单的打开/关闭状态，支持点击外部自动关闭（对应 HTML 的 `toggleDropdown` / `closeAllDropdowns` / `document.addEventListener`）。

```typescript
interface UseTrashDropdownReturn {
  openDropdownId: string | null;
  toggleDropdown: (dropdownId: string) => void;
  closeAllDropdowns: () => void;
}
```
