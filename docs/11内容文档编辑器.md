# 内容文档编辑器 — TipTap 迁移

> 从 `contentEditable` + `document.execCommand` 迁移到 TipTap v3 富文本编辑器。
> 依赖已安装（`@tiptap/react` + 5 个官方扩展 + `@floating-ui/dom`）。
> 更新时间：2026-06-02
> 状态：✅ 已完成

---

## 一、迁移目标

| 维度 | 现状 | 目标 |
|------|------|------|
| 编辑器引擎 | `contentEditable` div + `innerHTML` 读写 | TipTap v3 `useEditor()` |
| 浮动工具栏 | 自研 `useTextSelection` + `document.execCommand` | TipTap `BubbleMenu` 组件 + 链式命令 |
| 标题编辑 | 独立 `contentEditable` h1 | 保留独立 h1（不纳入 TipTap） |
| 内容存储 | HTML 字符串 → `editorStore.content` | 不变，`editor.getHTML()` 返回 |
| 自动保存 | 2s 延迟 `debounce` | 不变，TipTap `onUpdate` 替代 `onInput` |
| 只读模式 | `dangerouslySetInnerHTML` | TipTap `editable={false}` + `content` prop |
| 气泡菜单可见范围 | `[data-editor-body]` 选择器 | TipTap `BubbleMenu` 自动限定在编辑器内 |

## 五、EditorBody 实现细节

### 5.1 扩展配置

```ts
import { useEditor, EditorContent } from "@tiptap/react"
import { BubbleMenu } from "@tiptap/react/menus"
import StarterKit from "@tiptap/starter-kit"
import Image from "@tiptap/extension-image"
import Placeholder from "@tiptap/extension-placeholder"
import Table from "@tiptap/extension-table"

const extensions = [
  StarterKit.configure({
    heading: { levels: [1, 2, 3] },
  }),
  Image,
  Placeholder.configure({ placeholder: "开始写点什么..." }),
  Table.configure({ resizable: true }),
]
```

### 5.2 文档切换防抖

使用 `isSettingContent` ref 防止 `onUpdate` 在程序化 `setContent()` 时触发：

```ts
const isSettingContent = useRef(false)

// onUpdate guard
onUpdate: readOnly ? undefined : ({ editor }) => {
  if (isSettingContent.current) return
  setContent(editor.getHTML())
}

// 文档切换
useEffect(() => {
  if (!editor || readOnly) return
  isSettingContent.current = true
  editor.commands.setContent(content)
  requestAnimationFrame(() => {
    isSettingContent.current = false
  })
}, [selectedNodeId])
```

### 5.3 BubbleMenu 组件

完全内嵌在 `EditorBody` 中，不再作为独立组件暴露。

```tsx
{editor && (
  <BubbleMenu
    editor={editor}
    options={{
      placement: "top",
      offset: { mainAxis: 8 },
    }}
  >
    {/* 工具栏按钮: Bold / Italic / Strike / Link / Code */}
  </BubbleMenu>
)}
```

**UI 样式完全保留**：深色背景 `bg-[#1e2522]`、圆角 `rounded-xl`、图标尺寸等与旧版一致。

### 5.4 只读模式

```ts
const editor = useEditor({
  extensions,
  content: readOnly ? (externalContent ?? "") : content,
  editable: !readOnly,
  onUpdate: readOnly ? undefined : onUpdateHandler,
})

// 外部内容变化时同步
useEffect(() => {
  if (!editor || !readOnly || externalContent === undefined) return
  editor.commands.setContent(externalContent ?? "")
}, [externalContent, editor, readOnly])
```

`TrashDocumentViewer` 接口完全兼容，无需任何改动。

### 5.5 样式衔接

在 `globals.css` 中追加了 ProseMirror 专属样式：

```css
.prose-custom .ProseMirror {
  outline: none;
  min-height: 400px;
}

.prose-custom .ProseMirror p.is-editor-empty:first-child::before {
  content: attr(data-placeholder);
  float: left;
  color: #9ca3af;
  pointer-events: none;
  height: 0;
}

.prose-custom.read-only .ProseMirror {
  min-height: 200px;
}
```

**注意**：不违反 [09UI定制.md](./09UI定制.md)，只是追加选择器让现有 `prose-custom` 排版样式对 TipTap 内部 DOM 生效，不改变任何颜色/间距/字号。

## 七、API 对照表（已实现）

| 操作 | `contentEditable` 旧 API | TipTap 新 API |
|------|--------------------------|---------------|
| 获取内容 | `editorRef.current.innerHTML` | `editor.getHTML()` |
| 设置内容 | `editorRef.current.innerHTML = html` | `editor.commands.setContent(html)` |
| 加粗 | `document.execCommand('bold')` | `editor.chain().focus().toggleBold().run()` |
| 斜体 | `document.execCommand('italic')` | `editor.chain().focus().toggleItalic().run()` |
| 删除线 | `document.execCommand('strikeThrough')` | `editor.chain().focus().toggleStrike().run()` |
| 插入链接 | `document.execCommand('createLink')` | `editor.chain().focus().setLink({ href }).run()` |
| 行内代码 | `document.execCommand('insertHTML')` 包裹 `<code>` | `editor.chain().focus().toggleCode().run()` |
| 输入事件 | `onInput` → `e.target.innerHTML` | `onUpdate` → `editor.getHTML()` |
| 选区位置 | 自研 `useTextSelection` | TipTap `BubbleMenu` 自动计算 |
| 撤销 | 浏览器默认 | StarterKit `history` 扩展，`editor.chain().undo().run()` |
| 重做 | 浏览器默认 | StarterKit `history` 扩展，`editor.chain().redo().run()` |
| 只读模式 | `dangerouslySetInnerHTML` | `editable: false` + `setContent()` |
| 编辑器实例 | `useRef<HTMLDivElement>` | `useEditor()` 返回的 `Editor` 对象 |

## 八、数据流（迁移后不变）

```
EditorBody onUpdate
  → editor.getHTML()                          ← TipTap 输出
    → editorStore.setContent(html)            ← 不变
      → scheduleSave(2s)                      ← 不变
        → performSave()
          → editorService.saveContent(tree, nodeId, content, wordCount)
            → fileTreeRepo.setTree(newTree)   ← 不变
```

内容格式全程保持 HTML 字符串，与迁移前完全兼容。已保存到 mock 数据中的旧 HTML 内容无需任何转换即可被 TipTap 正确渲染。
