## 二、认证页模块

### 2.1 路由与入口

```
/ → page.tsx (LandingPage)
/login → (auth)/login/page.tsx → <AuthPage />
/logout → 调用 useAuth().logout() → push /login
```

### 2.2 组件树

```
app/(auth)/layout.tsx
  └── AuthLayout (已登录→跳转 /workspace)
        └── children

AuthPage (auth-page.tsx)
  ├── AuthCard
  │     ├── TabSwitcher (登录/注册切换)
  │     ├── AuthForm (表单: email + password + rememberMe | fullName)
  │     │     ├── FormInput × 2~4
  │     │     ├── Checkbox (仅登录)
  │     │     └── SubmitButton
  │     └── SocialLogin (仅登录, 分割线 + Google/Apple)
  └── (错误提示区)
```

### 2.3 数据流

```
[UI] AuthForm.handleSubmit()
  → auth-page.handleLoginSubmit({ email, password, rememberMe })
    → useAuth().login(email, password)
      → auth-service.validateLogin(email, pw)
        → userRepo.MOCK_USERS.find(...)  ← Repository
      → auth-service.createSession(user, rememberMe)
        → sessionRepo.createSession(user)  ← Repository
          → data/session-mock: localStorage.setItem()
      → setUser(user)

[UI] AuthForm.handleSubmit()
  → auth-page.handleRegisterSubmit({ fullName, email, password })
    → useAuth().register(payload)
      → auth-service.validateRegister(payload)
        → userRepo.findUserByEmail()  ← Repository
        → userRepo.addUser(newUser)   ← Repository
      → sessionRepo.createSession(user)  ← Repository
      → setUser(user)
```

|----|------|------|

| UI | `components/auth/auth-card.tsx` | 卡片容器 |

| 布局 | `app/(auth)/layout.tsx` | 已登录自动跳转 /workspace |
| 状态 | `lib/auth-context.tsx` | useAuth() → login/register/logout |
| 状态 | `hooks/use-auth-tab.ts` | Tab 切换动画 |
| 业务 | `lib/services/auth-service.ts` | validateLogin / validateRegister / session |
| 业务 | `lib/services/userService.ts` | updateUserInfo / validateOldPassword |

## 三、应用主页模块 (/workspace)

### 3.1 路由与入口

```
/workspace → (main)/workspace/page.tsx → WorkspacePage
  └── AuthGuard 保护（未登录 → /login）
```

### 3.2 组件树

```
WorkspacePage (page.tsx)
  ├── SidebarLayout (整体布局)
  │     ├── 侧边栏 (aside)
  │     │     ├── UserSection (用户头像 + 折叠按钮 + 下拉菜单)
  │     │     ├── SearchBar (搜索输入)
  │     │     ├── FileTree (文件树)
  │     │     │     └── TreeItem × N (inline rename + TreeItemMenu)
  │     │     └── SidebarFooter (精选 & 垃圾箱入口)
  │     │
  │     ├── 折叠时顶部栏: [☰] + TopBarBreadcrumb
  │     │
  │     └── 内容区 (children)
  │           ├── currentView === "editor" ?
  │           │     DocumentEditor
  │           │       ├── EditorHeader (面包屑 + SaveStatusBadge + TopBarDropdown)
  │           │       ├── EditorBody (contentEditable 标题 + prose-custom 正文)
  │           │       └── BubbleMenu (选中文本浮动工具栏)
  │           │
  │           └── currentView === "folder" ?
  │                 FolderView
  │                   ├── Breadcrumb (面包屑)
  │                   ├── FolderListHeader (表头)
  │                   └── FolderListItem × N (点击进 folder 或 editor)
```

### 3.3 状态管理核心

```
useFileTreeStore (Zustand)
  state:  tree / selectedNodeId / expandedIds / currentView / creatingNodeId
  actions: selectNode | toggleExpand | createFile | createFolder |
           commitRename | cancelCreate | deleteNode | setTree | restoreFromTrash
  derived: useBreadcrumb | useFolderItems | useFolderName | useDocumentInfo | useSelectedNode

useEditorStore (Zustand)
  state:  content / title / wordCount / isSaved / isDirty
  actions: loadEditorFromTree | setContent | setTitle | saveNow | cancelAutoSave
```

### 3.4 数据流（核心操作）

#### 选中节点

```
TreeItem (UI) onClick
  → useFileTreeStore().selectNode(id)
    → findNode(tree, id)  ← fileTreeService
    → set({ selectedNodeId, currentView: node.type === "folder" ? "folder" : "editor" })
  → useEffect: loadEditorFromTree(id)  ← editorStore
      → findNode(tree, id) → set({ content, title, wordCount })
```

#### 创建文件/文件夹

```
TreeItemMenu (UI) [新建文档/新建文件夹]
  → useFileTreeStore().createFile(parentId)
    → createNode(tree, parentId, "file")  ← fileTreeService (纯函数)
      → findNode → generateUniqueName → generateId → formatTimestamp
      → 返回 { newTree, createdNode }
    → fileTreeRepo.setTree(newTree)  ← Repository
    → set({ tree: newTree, selectedNodeId, currentView: "editor" })
```

#### 重命名

```
FolderListItem (UI) [⋮ → 重命名] → window.prompt
  → useFileTreeStore().commitRename(id, name)
    → renameNode(tree, id, name)  ← fileTreeService
      → getSiblingNames → validateName
    → fileTreeRepo.setTree(newTree)
    → set({ tree: newTree })
```

#### 删除到垃圾箱

```
TreeItemMenu (UI) [⋮ → 删除]
  → useFileTreeStore().deleteNode(id)
    → moveToTrash(tree, id)  ← fileTreeService
      → deleteNode → countDescendants → 生成 TrashItemData
    → trashRepo.addToTrash(trashItem)  ← Repository
    → fileTreeRepo.setTree(newTree)    ← Repository
    → set({ tree: newTree })
```

#### 编辑器自动保存

```
EditorBody (UI) onInput → setContent(html)
  → scheduleSave() (2s 延迟)
    → performSave()
      → saveContent(tree, nodeId, content, wordCount)  ← editorService
      → fileTreeRepo.setTree(result.tree) ← 或通过 store.setTree
```

### 3.5 依赖清单

| 层 | 文件 | 职责 |
|----|------|------|
| 页面 | `app/(main)/workspace/page.tsx` | 薄编排：视图切换 |
| UI | `components/workspace/content/DocumentEditor.tsx` | 编辑器容器 |
| UI | `components/workspace/content/FolderView.tsx` | 文件夹视图 |
| UI | `components/workspace/editor/editor-body.tsx` | 编辑器正文（含 readOnly 模式） |
| UI | `components/workspace/editor/editor-header.tsx` | 编辑器顶栏 |
| UI | `components/workspace/layout/SidebarLayout.tsx` | 整体布局 |
| UI | `components/workspace/layout/TopBarBreadcrumb.tsx` | 折叠时面包屑 |
| UI | `components/workspace/common/Breadcrumb.tsx` | 面包屑导航 |
| UI | `components/workspace/common/SaveStatusBadge.tsx` | 保存状态 |
| UI | `components/workspace/folder/` | 文件夹列表组件 |
| UI | `components/workspace/sidebar/` | 侧边栏组件 |
| 状态 | `store/fileTreeStore.ts` | 文件树状态 |
| 状态 | `store/editorStore.ts` | 编辑器状态 (2s 自动保存) |
| 业务 | `lib/services/fileTreeService.ts` | 文件树纯函数集合 |
| 业务 | `lib/services/editorService.ts` | 保存内容 + 更新标题 |
| 数据 | `repositories/` → `fileTreeRepo` + `trashRepo` + `workspaceRepo` | 数据访问 |
| 数据 | `data/mockFileTree.ts` | 初始文件树 |

### 3.6 两个 Store 的协作关系

```
editorStore.setContent(title)
  → scheduleSave(2s)
    → performSave()
      → useFileTreeStore.getState()  ← 跨 store 读取 tree/selectedNodeId
      → saveContent(tree, nodeId, content, words)  ← editorService
      → useFileTreeStore.getState().setTree(result.tree)  ← 写回 fileTree

workspace/page.tsx:
  useEffect → loadEditorFromTree(selectedNodeId)
    → useFileTreeStore.getState().tree  ← 跨 store 读取
    → findNode(tree, selectedNodeId)    ← fileTreeService
    → set({ content, title, ... })      ← 写入 editorStore
```

## 四、垃圾箱页模块 (/trash)

### 4.1 路由与入口

```
/trash → (main)/trash/page.tsx → TrashPage
  └── AuthGuard 保护
  └── SidebarLayout isTrashActive
```

### 4.2 组件树

```
TrashPage (page.tsx)
  ├── previewItem ? TrashDocumentViewer : 列表视图
  │
  ├── 列表视图:
  │     ├── TrashHeader (标题 + 清空按钮)
  │     ├── 表头行
  │     └── TrashItem × N
  │           ├── 文件名 + 图标
  │           ├── 数量 / 最近编辑 / 创建时间
  │           └── FloatingMenu (多选 / 恢复 / 永久删除)
  │
  ├── 预览视图 (TrashDocumentViewer):
  │     ├── 操作栏: [← 返回回收站] [恢复] [永久删除]
  │     └── EditorBody(readOnly, externalTitle, externalContent)
  │           ├── <h1> 标题
  │           └── prose-custom 只读正文
  │
  └── FloatingBatchBar (批量操作浮动栏, 仅批量模式)
```

### 4.3 状态管理

```
useTrashStore (Zustand)
  state:  items / isBatchMode / selectedIds
  actions: refresh | enterBatchMode | exitBatchMode |
           toggleSelectItem | toggleSelectAll |
           restoreItem | deletePermanently | emptyTrash |
           batchRestore | batchDelete
```

### 4.4 数据流

#### 查看列表

```
trash/page.tsx
  → useTrashStore().items  (初始化时 trashRepo.getTrash())
```

#### 点击预览文件

```
TrashItem (UI) onClick (type==="file" && !isBatchMode)
  → onPreview({ id, name, content, ... })
    → setPreviewItem(item)
      → TrashDocumentViewer(item)
        → EditorBody(readOnly, externalTitle=item.name, externalContent=item.content)
```

#### 恢复文件

```
操作栏 [恢复] 或 FloatingMenu [恢复]
  → handleRestore(id)
    → setPreviewItem(null)  (如果从预览触发)
    → restoreFromTrash(id, trashStore.restoreItem)
      → restoreItem(item, tree)  ← trashService
        → 重建 TreeNode → push 到父节点
        → trashRepo.removeFromTrash(id)
      → fileTreeStore.setTree(newTree)
    → router.push("/workspace")
```

#### 永久删除

```
操作栏 [永久删除] 或 FloatingMenu [永久删除]
  → handleDeletePermanently(id)
    → trashStore.deletePermanently(id)
      → deletePermanently(id)  ← trashService
        → trashRepo.removeFromTrash(id)
    → set({ items: trashRepo.getTrash() })
```

### 4.5 依赖清单

| 层 | 文件 | 职责 |
|----|------|------|
| 页面 | `app/(main)/trash/page.tsx` | 列表/预览双视图切换 |
| UI | `components/workspace/trash/TrashHeader.tsx` | 标题 + 清空 |
| UI | `components/workspace/trash/TrashItem.tsx` | 列表行 + FloatingMenu |
| UI | `components/workspace/trash/TrashDocumentViewer.tsx` | 预览：操作栏 + EditorBody |
| UI | `components/workspace/trash/FloatingBatchBar.tsx` | 批量操作浮动栏 |
| 状态 | `store/trashStore.ts` | 垃圾箱状态 |
| 状态 | `store/fileTreeStore.ts` | 恢复需写回文件树 |
| 业务 | `lib/services/trashService.ts` | restoreItem / deletePermanently / emptyTrash |
| 数据 | `repositories/` → `trashRepo` + `workspaceRepo` | 数据访问 |
| 数据 | `data/mockTrash.ts` | 初始垃圾箱数据 |

### 4.6 视图切换逻辑

```
previewItem === null  → 渲染列表视图（原有完整逻辑）
previewItem !== null  → 渲染 TrashDocumentViewer (替换列表区)

所有操作都先 setPreviewItem(null) 清除预览状态:
  恢复   → setPreviewItem(null) + restore + router.push("/workspace")
  永久删除 → setPreviewItem(null) + deletePermanently
  返回列表 → setPreviewItem(null)
```

## 五、用户设置模块

### 5.1 入口

```
UserSection (侧边栏头像)
  → UserMenu 下拉
    → [账号设置] → AccountSettingsDialog (Modal)
    → [会员订阅] → MembershipDialog (Modal)
    → [退出登录] → logout() → push /login
```

### 5.2 组件树

```
UserSection (侧边栏)
  └── UserMenu (头像 + 下拉菜单)
        └── AccountSettingsDialog (Modal, 账号设置)
              ├── 左侧 Tab 列表: [账号设置] [通知音效] [设备管理]
              └── 右侧内容:
                    ├── AccountTab (修改资料/密码/手机/邮箱/注销)
                    ├── NotificationsTab (通知开关)
                    └── DevicesTab (已登录设备列表)

AccountTab:
  ├── 个人信息 → Modal (修改姓名/头像)
  ├── 会员信息 → MembershipDialog (升级弹窗)
  ├── 手机号 → Modal (修改手机号)
  ├── 邮箱 → Modal (修改邮箱)
  ├── 密码 → Modal (修改密码, 需验证原密码)
  └── 注销账号 → Modal (输确认文字 + 勾选)
```

### 5.3 数据流

#### 修改个人信息

```
AccountTab handleSaveProfile
  → validateFullName(name)  ← utils/validators.ts
  → updateUser({ fullName, avatar })  ← useAuth().updateUser()
    → userService.updateUserInfo(user, updates)
      → { ...user, ...updates }
      → userRepo.updateUserInStorage(updated)  ← Repository
        → data/mockUsers.updateUserInStorage()
    → setUser(updated)
```

#### 修改密码

```
AccountTab handleSavePassword
  → validateOldPassword(user, oldPw)  ← userService (纯比较)
  → validatePasswordStrength(newPw)  ← utils/validators
  → validateConfirmPassword(newPw, confirm)
  → updateUser({ password: newPassword })
```

#### 注销账号

```
AccountTab handleDeleteAccount
  → logout(true)  ← useAuth().logout(clearAll)
    → clearAllUserData()  ← userService
      → sessionRepo.clearAllData()  ← Repository
        → localStorage.clear()
    → setUser(null)
  → router.push("/")
```

## 七、修改指南

### 7.1 新增路由页面

1. 在 `app/(main)/` 下创建 `page.tsx`
2. 需要认证保护 → 放在 `(main)/` 路由组内
3. 需要侧边栏 → 用 `SidebarLayout` 包裹 children
4. 按八层架构严格单向依赖

### 7.2 新增数据操作

1. `types/` → 定义接口
2. `data/` → Mock 数据 + 存取函数
3. `repositories/types.ts` → 新增 Interface
4. `repositories/mock-xxx.ts` → Mock 实现
5. `repositories/index.ts` → 注册实例
6. `lib/services/` → 业务逻辑纯函数
7. `store/` → Zustand store
8. `components/` → UI

### 7.3 修改已有功能

- 改 UI → 只改 `components/` 下的对应文件
- 改校验逻辑 → `utils/validators.ts` 或 `lib/services/`
- 改初始数据 → `data/` 下的对应 mock 文件
- 改数据结构 → 同时改 `types/` 和 `data/` 和 `repositories/mock-xxx.ts`
